5. 推廣 Linux

  • 把你個人使用 Linux 的經驗 (不論好壞) 與大家分享。 大家都知道軟體都有臭蟲與缺點。 如果我們只提出 Linux 的優點, 那就是不誠實了。 我最喜歡告訴別人這個經驗: 我三年來只重新開機四次, 其中有三次是按照預定計劃重新開機的。
  • 如果有人有 Linux 可能可以解決的問題, 主動提供他 (她) 相關資訊, 例如網址, 雜誌, 文章, 書籍, 顧問 ... 等。 如果你自己不曾真的 照著你提供的建議去解決過類似的問題, 也應該說明白。 幫助 Linux 新手上手。
  • 如果你可以演講介紹 Linux, 請在 Linux Speakers Bureau 登記。
  • 幫助新手上路。 追蹤協助, 讓他們使用 Linux 的方式更有效率。
  • 還有人以為 Linux 還停留在純文字模式的時代。 把 Gimp 這類圖形軟體秀給他們看。
  • 「週行一善」 -- 從 newsgroup 中 Linux 新手的問題裡找一個 你自己覺得最有挑戰性而能力可及的問題來回答。 說不定你是唯一能夠解決問題的人。 此外, 這也是你學習的機會。 然而如果你不是很有把握, 應就教於有把握的人。
  • 與小型的軟體公司聯絡, 主動要求介紹 Linux 的機會。
  • 如果有機會的話, 對你公司裡的資訊技術部門介紹 Linux。
  • 參與 NetDay 之類的社群活動。 當然你的首要任務必須配合該活動的宗旨, 但也無妨藉機會推廣 Linux。
  • 一定要從你 "推銷" Linux 的對象他的角度來思考問題。 一位決策人員要考慮的問題包含了: 技術支援, 穩定性, 可交換性 (interoperability) 與花費。 這其中花費往往是最不重要的因素。
  • 很多人往往擔心 Linux 沒有技術支援。 其實像是 Caldera, Cygnus Solutions, Red Hat, 及 S.u.S.E. 等公司就對各自的 distribution 提供部分或全面的技術支援。 此外, Linux Consultants HOWTO 裡列舉出許多提供技術支援的公司。 當然, 最好的技術支援也經常來自 comp.os.linux 及 linux 底下的新聞群組。
  • 強調 開放原始碼軟體 的研發方式, 提供了系統分析師, 程式設計師, 作家, 測試人員, 使用者等人員一個開放自由的合作環境, 也因而產生了可靠的產品及豐富的文件, 像 Apache, GNU Emacs, Perl and the Linux kernel 等等都是很成功的例子。
  • 讓大家知道你也在用 Linux! 請到 Linux Counter 註冊。
  • 把推廣 Linux 的成功案例報告到 Linux International ( li@li.org) 之類的組織去。
  • 如果你有一些已經不太需要的 Linux 光碟片和書籍, 就把它捐出去, 例如送給對 Linux 有興趣的人, 圖書館, 或是學校的實驗室等等。 附有光碟片的書最適合捐給圖書館了。 但是要注意捐出來給大眾用的光碟片不可以違反著作權聲明。 另外, 要告訴圖書館管理員, 光碟片上的資料是可以合法自由拷貝的。 記得隨後再去確認一下你捐出來的書確實有上架。
  • 購買 Linux 上的應用軟體的書籍時, 請優先考慮該軟體作者所寫的書。 這樣做可以為該軟體作者帶來金錢的回饋。 <-- Need to fix or change the Powered by Linux text -->
  • 鼓勵 Linux 的用戶分享他們的 「我們用 Linux!」 標誌/圖案, 也鼓勵大家把 Linux, Apache, GNU, Perl ... 這些軟體的標誌/圖案大方地秀出來。
  • 更多的參與! 如果你自己因為 open-source software 而受益, 請考慮以下列方式回饋:
    • 做詳盡的臭虫回報
    • 撰寫文件
    • 製作標誌/圖案/... 等等藝術創作
    • 貢獻管理長才
    • 建議新的/改善的功能
    • 提供技術支援
    • 貢獻軟體
    • 捐獻硬體設備
    • 捐獻金錢
    在 Linux Documentation Project 可以找到許多 Linux 相關計劃的 索引
  • 最後, 請不要忘記我們還有很多比「選擇一個作業平臺」更重要的事情要操心。
