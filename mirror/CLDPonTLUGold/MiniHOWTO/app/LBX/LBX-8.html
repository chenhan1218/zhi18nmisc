8. 如何使 LBX 開始工作？

OK，接下來的步驟都很簡單。 將下列 LOCAL 以及 REMOTE 用你本端以及遠端的主機名稱取代。 ( 千萬不要搞混了 )On LOCAL:

  1. 啟動你的 X server。
  2. 告訴你的 X server 哪些遠端系統是被允許存取的。 使用 host-list 的方法，鍵入 xhost + REMOTE。如果你要使用 xauth 你需要作更多事情；請參閱 man xauth(1) 以獲得更多的資訊。 如果你不熟悉 X 遠端的存取，那麼你需要參閱 Remote X Apps Mini-HOWTO 。

On REMOTE:

  1. 起始 lbxproxy 並告訴它傳送訊息到本端的 X server，如下︰ 
      $ lbxproxy -display LOCAL:0 :1 &
    這告訴 lbxproxy 在遠端系統使用 display :1 ；如果系統大於 1 的 display 已經在使用。你可以使用 :2 之類的設定取代。
  2. 設定你的 DISPLAY 環境變數，使此環境變數和你 lbxproxy 所設定的相同。取代平常的 DISPLAY 設定 
      $ DISPLAY=:1
  $ export DISPLAY
    如果你使用 csh 之類的，以下可能是你所需要的︰ 
      % setenv DISPLAY :1
  3. 如果你使用 xauth 你需要確定你的 cookie 正確的運作。你可以從 Remote X Apps Mini-HOWTO 獲的更多的資訊。
  4. 執行你的 X 程式！

接下來就完成了；所有在 Display 環境變數為:1 的 X 程式都會在都會使用 LBX。當然，你沒有辦法起始一個 X 程式，其 Display 的環境變數是指向 LOCAL:0 以及在同一個時間下執行。
