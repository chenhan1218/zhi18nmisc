文件的狀態

這份文件詳細說明了網際網路社群的網際網路標準追蹤協定，並對於改善網際網路標準請求討論上的建議。若想知道標準化的情況與此協定(LDAPv3) 的狀態，所以請參考現在編輯的"網際網路官方協定標準"(STD1)。散佈這份文件並沒有什麼限制條件。

摘要

這份文件詳細介紹一組由輕量目錄存取協定第三版(LDAPv3)所組成的 RFC，並且會處理由 RFC 2251 到 2256 間 "IESG 的注意事項"。

1. 背景資料與動機

輕量目錄存取協定第三版(LDAPv3)的規格書在名義上可以由八份 RFC 組成，而這八份 RFC 依照時間的區隔可分成兩個子集。首先是 RFC 2251 到 2256，而後是 RFC 2829 與 RFC 2830。


RFC 2251 到 2256 並沒有強制規定要實作任何功能強大的認證機制，因此發佈文件時附上 "IESG 注意事項" 來阻止實作與佈署 LDAPv3 客戶端或伺服端實作更新功能，直到發佈 LDAPv3 強制規定認證的建議標準為止。


隨後發佈的 RFC 2829 是回答 IESG 注意事項。


此份文件的用意在說明 LDAPv3 是由那些 RFC 組成的，並藉由 RFC 2829 的內容來正式處理 IESG 注意事項。

2. 輕量目錄存取協定第三版的規格書

這份文件詳細說明輕量目錄存取協定第三版(LDAPv3)是由一組九份的 RFC 組成：

  • [RFC2251] - 輕量目錄存取協定第三版：屬於不安全的 LDAP 規格書
  • [RFC2252] - 輕量目錄存取協定第三版：屬性語法定義
  • [RFC2253] - 輕量目錄存取協定第三版：以 UTF-8 字串表達已區分的名稱
  • [RFC2254] - 以字串表達 LDAP 搜尋過濾器
  • [RFC2255] - LDAP URL 格式
  • [RFC2256] - 簡介 X.500(96)使用者綱要，為了使用 LDAPv3
  • [RFC2829] - LDAP 的認證方法
  • [RFC2830] - 輕量目錄存取協定第三版：傳輸層安全的延伸功能

以及此份 RFC 3377 文件。


術語 "LDAPv3" 常常是以非正式的用法在上方這一組 RFC 代表此通訊協定。然而，LDAPv3 協定集，就如定義在這裡的，應該正式被其它文件所識別經由正式的參考到這份文件。

3. 處理在 RFC 2251 到 2256 間的 "IESG 注意事項"

IESG 批準發佈 RFC2251 到 2256 時在每份文件附上 IESG 注意事項。注意事項開始於這樣的文字：


這份文件介紹一種目錄存取協定以提供讀與更新的存取。更新存取需要安全認證，但這份文件不強制規定實作任何功能強大的認證機制。


而注意事項終止於這樣的文字：


阻止實作者佈署 LDAPv3 客戶端或伺服端實作其更新功能，直到發佈在 LDAPv3 強制規定認證的建議標準。


[RFC2829] 是解釋"在 LDAPv3 中強規定認證的建議標準" 會在注意事項說明。因此會處理 [RFC2251]、[RFC2252]、 [RFC2253]、[RFC2254]、[RFC2255] 的 IESG 注意事項。

4. 安全考量

雖然前節的 IESG 注意事項與安全有關，但此份文件不直接討論安全問題。


若想進一步了解 LDAPv3 的安全問題，請參考相關文件，尤其是 [RFC2829]、[RFC2251]與 [RFC2830]。
