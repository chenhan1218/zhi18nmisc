4. LUG 做些什麼?

隨著區域的不同, LUG 的目標也各有很大的差異。 並沒有所謂的「LUG 規範」 (本文也不是)。 沒有一個「中央政府」在統治 Linux； 同樣地，區域性的 LUG 也都是獨立自主的。

不過我們倒是可以指出幾項常見的區域性 LUG 核心工作：

  • 推廣
  • 教育
  • 技術支援
  • 公關

每個區域性的 LUG 將上述工作與其特性結合，來滿足它成員獨特的需求。

4.1 推廣 Linux 系統

對電腦使用者來說，想要推廣 Linux 這種想法就像吃飯和睡覺一樣自然。 當你發現有樣東西功能很好，很自然的就會想要告訴所有的人。 各地 LUG 在推廣 Linux 的過程中所扮演的角色非常重要， 特別是因為 Linux 雄厚的商業化潛力一直都未受到一般人正視。 在這波 Linux 運動中，有電腦雜誌對 Linux 作正面的報導固然值得喝采； 即使你我只是一個使用過 Linux 之後感到很滿意的個人， 我們能夠告訴我們的朋友、同事、上司、部屬, 也都是好事。

鼓吹使用 Linux 的行動可以做得很有效率，也可能流於吹毛求疵。 Linux 使用者應該要保持推廣的活力，但這同時必須注意到推廣的方式。 以使社會對這個產品本身、它背後的研發者、以及它的使用者產生正面的印象。 在 Linux Documentation Project 中的 Linux Advocacy mini-HOWTO 裡對這點有更詳細的說明，在這裡我們只強調區域性 LUG 在推廣 Linux 這項工作上扮演著舉足輕重的角色。

也許有一天 Linux 終將受到它應得的肯定， 也許有一天“沒有人會因為使用 Linux 而丟掉工作”終將成為事實， 那麼推廣 Linux 就不再是一件這麼重要的事情了。 但是在這天來到之前, 區域性的 LUG 在推廣 Linux 這項工作上都將扮演不可或缺的角色。 因為這樣的推廣活動背後的動機是善意的、沒有利害關係的， 而且是由一個長期性的組織在支持。 經由 LUG 知道 Linux 的人在運用 Linux 這個過程當中已經佔了先機： 知道有人願意幫助安裝、設定、甚至管理 Linux 機器。

這些人比起經由電腦雜誌獲得 Linux 資訊、但卻求助無門的讀者要幸運的多了。 後者可能不知道要找誰幫忙他安裝、使用、學習 Linux。

從這個角度看，由區域性的 LUG 來推廣 Linux 更有效率， 更有後盾，而且可以提供免費的服務，因此區域性 LUG 加入這項活動是有必要的。

4.2 Linux 教育

區域性的 LUG 不僅可以從事推廣 Linux 的工作， 更可以對它的成員乃至該地區的電腦使用人口提供教育訓練， 協助他們使用 Linux 及相關的自由軟體。 我個人認為教育訓練可以說是 LUG 最重要的使命。 當然, 先前也說過, 每個 LUG 可以自由決定他們的運作重心； 但是我相信在這波 Linux 運動中，LUG 最能夠發揮影響力的地方就在於教育訓練了。

有些 LUG 有鑒於一般的 Linux 使用者缺乏學習相關技術的機會， 因而攬下了培訓工作。 固然有越來越多的大專院校注意到 Linux 的高品質與低價位， 因而開始用它來教 UNIX 系統； 但是仍舊有一些 Linux 的使用者或者缺乏意願，或者沒有機會修這樣的正課。 這些使用者可以從 LUG 處學習到許多進階的電腦技能，諸如 UNIX 系統管理， 系統程式設計，Internet 及 Intranet 環境的架設與管理等等。

諷刺的是，許多 LUG 其實是在分擔大公司的培訓工作。 每多一位 Acme Corp 的員工因為參與 LUG 的活動而增長了電腦技能， Acme Corp 就省下一份訓練員工的資源或經費。 當然在家裡使用、管理一部跑 Linux 的 PC 跟在大公司等地方多人使用的大電腦上的環境不太一樣。 但是比起使用 Windows 95，使用 Linux 所獲得的知識及經驗不知道要多上多少倍。 而且隨著 Linux 在及時性及其他尖端技術方面的進步， 它與 "真正的" UNIX 之間原本就不明顯的差異也將越來越模糊。

今日的資訊科技在全球的經濟中扮演著越來越重要的角色， 而 LUG 提供這樣的教育訓練機會不僅可以訓練人員， 更是一種以社區服務方式推廣 Linux 的機會。 以美國大城市為例，LUG 可以將 Linux 帶入學校、中小企業、社區、社團組織等地方。 這麼作有推廣 Linux 的效果，讓一般大眾得以借由 Linux 學習 UNIX 系統。 如果這些機構決定架設網站以提高知名度或提供內部成員上線服務， LUG 就有機會進一步將這樣一個自由作業系統介紹給社區大眾，這也是一種社區服務。 這種大方貢獻自己知識力量的精神正是 Linux 以及自由軟體界的特點之一， 雖然不是每位 Linux 使用者都像 Linux Torvalds 一樣會寫程式， 我們總可以做一些我們能力所及的事來回饋其他 Linux 使用者、Linux 社會、甚至是我們所居住的社區。

這些機構正是 Linux 展現實力的最佳場所， 因為對他們來說這省下來的不僅僅是昂貴的使用許可權、升級、維護等等費用。 這些非營利機構往往樂於接受大公司淘汰下來的硬體。 Linux 對硬體的使用效率遠超過一般流行的作業系統， 就算在這些老舊的機器上一樣可以發揮功能。 越來越多人發現閑置在牆角養蚊子用的舊 486 只要裝上了 Linux， 其實還是可以做很多事。

Linux 的培訓工作對於一個 LUG 所要從事的其他活動 (尤其是下述的技術支援工作) 而言，具有加成的效果。 越成功的培訓將帶來越成功的技術支援。 LUG 當中有越多人有足夠的知識和技術，對外的技術支援就越輕鬆， 而支援的範圍也可以越大。 如果 LUG 能提供更多的培訓機會給初學者，LUG 本身的組織與力量也就越大。 反之，如果 LUG 只專注在技術支援而忽略了培訓， 那麼這個團體的成長自然會遇到瓶頸。 如果新的 Linux 使用者無法從 LUG 處得到適當的援助， 他們很快就會離開 LUG。 如果 LUG 內部有較多的人力投入對其他使用者的教育訓練工作， 那麼 LUG 自然就有更大的成長空間。 Linux 的培訓在這裡扮演了非常重要的角色：它將新手轉變為老手。

Linux 是自由軟體的一角，服務性質的 Linux 教育訓練因而也是一個向大眾介紹自由軟體的機會。 LUG 可以將視野放寬，除了介紹 Linux 的使用之外，也介紹其他相關的自由軟體。 例如許多的 GNU 應用程式、GNU 工具軟體、Apache 網路伺服器、XFree86、TeX、LaTeX 等等。 在 Linux 底下的自由軟體數量種和類都相當的多。

最後，Linux 可以說是一個在文件製作上自給自足的系統， 如果我們自己不寫手冊，大概就沒有人會替我們寫了。 所以應該要讓 LUG 所有的成員都知道 Linux Documentation Project (譯註: 對臺灣而言，那就是 Linux 文件中譯計劃了。) LUG 可以提供映射 Linux 文件的服務，讓 LUG 成員及本地的 Linux 使用者方便取得文件。 將既有的 Linux 文件廣為宣傳，讓大家知道有這樣的資源可以使用， 這也是一件非常重要的事。 不只是 LDP，其他任何 Linux 的文件，例如 LUG 出版的技術報告。 學習手冊，區域性的常問問題集等等， 都可以經由comp.os.linux.announce公諸於世。 有一些區域性的 LUG 費盡心力製作 Linux 文件， 但卻由於宣傳不足，結果只有當地的 Linux 使用者知道有這些資料可以運用。 我們要讓世界上所有 LUG 的努力成果得以分享：往往某個 LUG 的成員的問題解答也就是世界上其他角落許多 LUGs 成員所需要的問題解答。

4.3 Linux 技術支援

對於一個求助無門的新手而言，最需要的就是當地 LUG 的支援了。 然而 LUG 可以提供的支援並不僅止於針對新手的技術支援。

LUG 可以提供下列的支援：

  • 使用者
  • 技術顧問
  • 公司行號, 非營利事業, 學校
  • Linux 運動

使用者

Linux 新手最常抱怨的就是不容易上手。 事實上這並不是 Linux 才有的問題，當代所有版本的 Unix 都一樣。 但是伴隨著艱辛的學習過程而來的，是對作業系統真正的瞭解與靈活駕馭的能力。 區域性的 LUG 可以幫助新手把學習的道路變得更舒坦。

事實上 Linux 新手或許自己不知道，他真正需要的不僅僅是技術支援而已。 Linux 與自由軟體的世界時時都有新的發展，而 LUG 正是這些資訊的傳遞管道。 Linux 背後不僅沒有一個掌控一切的官僚體系， Linux 資訊傳播的管道也不如其他作業系統多元化。 我們有諸如 Linux Journal 及 Linux Gazette 等定期刊物，但是新手往往不知道。 此外，由於這些是月刊，如果你想知道最新的修正公告。 新版的核心，安全問題等等, 這往往都有點過時。 這也正是區域性 LUG 可以幫得上忙的地方： 不論是新手或老手都可以從這裡獲得及時的資訊。

比方說吧，在一位新手還不知道他可以從 ftp.kernel.org 處取得最新的核心，也不知道 Linux Documentation Project 處的 HOWTO 文件往往比光碟上的資料還要新之前， 區域性的 LUG 或許就是新手最主要的及時資訊來源了。

其實 LUG 提供支援的對象並不僅止於新手， 進階的使用者或是老手如果能夠獲得及時的資訊、技巧、內幕消息等等， 也將獲益良多。 Linux 的內容實在太豐富了，即使是老手也可以經由參與 LUG 的活動而學到新的技術，像是學習過去不曾聽過的應用軟體， 或是複習好幾年都不曾用過的vi指令等等。

技術顧問

大家應該都會同意區域性的 LUG 當然要提供技術支援服務 Linux 新手。 要不然 LUG 要做什麼事呢? 另一方面，不論是全職或兼職的 Linux 技術顧問， 他們對 LUG 而言也扮演非常重要的角色。 LUG 其實也可以提供他們一些服務。

LUG 可以提供那些服務呢? 換個角度問：Linux 技術顧問們需要的是什麼呢? 他們需要顧客。 LUG 正是最佳的資訊交流中心，它可以將需要 Linux 技術支援的人轉介給提供 Linux 技術支援的人， 具體的方法我們等一下談；這裡要強調的是: LUG 很可以， 也應該要扮演好這個角色。 在這方面，Linux Consultants HOWTO 這份文件是一份相當重要的文件； 可惜並不是全世界的 (全職或兼職) Linux 顧問都列入其中.

在這樣的互惠關係中，技術顧問們可以提供技術與組織方面的經驗給 LUG； 而 LUG 可以提供技術顧問們客戶的來源。 至於使用者們呢，不論是新手或是玩家，也都一樣受惠： 他們如果有簡單的問題，可以從 LUG 處獲得免費的協助； 如果有比較複雜或是需要人力解決的問題， 則可以透過 LUG 從技術顧問處購買服務。

需要顧問或不需要顧問的要求有時分別不大：但在多數情況下卻有明確的不同。本地 LUG 不應背上唆使新手不尋求顧問提供支助的罪名--這種作法不但粗魯，而且不符合 Linux 的精神-- 因此 LUG 並沒有理由不協助拉攏需要專家指導的使用者和提供這種專家指導的專業人員 之間的聯系。

從 Martin Michlmayr 的 Linux Consultants HOWTO 上可以找到世界各地的顧問名單。

企業, 非營利事業, 學校

LUG 也可支助當地企業和機構。這種支助可分兩方面：首先，LUG 可以支助預備使用 Linux 滿足 其計算或信息技術需要的企業和機構，其次，LUG 也能夠支助開發 Linux、滿足 Linux 用戶 需要以及支持或安裝 Linux 的企業和組織。

LUG 向願意使用 Linux 作為其部分計算能力的本地企業提供的支助與提供給在自己家裡運行 Linux 的個人的支助并沒有很大不同。例如，以編輯 Linux 內核而言，為個人或為企業并無不同。 不過，支持使用 Linux 的企業可能得使 LUG 需要注意在 Linux 上運行的商業軟件，而不只關注免費 軟件。如果要繼續維持 Linux 的勢頭，使其成為一套可行的計算系統，就必需與願意將 Linux 當作市場上 可以生存的平台而發展軟件的商家合作。如果地區性的 LUG 能夠協助企業評估商用 Linux 軟件的優劣， 就會有更多的軟件廠商願意將 Linux 列入它們的發展計畫。

這引導我們提供第二種支助，也就是地區性的LUG對本地企業的支助。地區性的LUG可作為 交流中心，提供不易得到的信息資料。例如：

  • 哪一個本地 ISP 對 Linux 提供方便?
  • 是否有任何本地商家製作 Linux 電腦?
  • 本地何處可購得 Linux 光碟?

收集和提供這種資料不僅有助于本地 LUG 的會員，同時也協助本地對 Linux 友善的商家。這使 這些商家繼續對 Linux 友善。有時這甚至會造成一種競爭，使其他商家也競向對 Linux 表示友善。

自由軟件的發展

通過募捐也是支助 Linux 運動的一種方式。 Chris Browne 想得比任何人都多。他提出下列看法。

Chris Browne對自由軟件的看法

多方參與可使各種與 Linux 有關的組織提供財務方面的支助。隨著 multiple millions 個 Linux 使用者， 其中一部分有心人士完全有可能提供金錢支助。如果有一百萬名使用者，每名Linux使用者 捐助不是離譜的100美元（ $100 約等於 不 買新版視窗操作系統的費用）， 則可以收集到 一億 美元用于發展和改善 Linux 的工具和應用。

使用者團體可鼓勵會員資助各種“發展項目”。如果這種捐款可以“免稅”， 則應鼓勵會員將錢捐給社團，給予適當的稅務減免，捐款則可捐給其他組織。

在任何情況下，均可鼓勵Linux使用者協會的會員將錢直接捐給他們個人希望支持的項目或努力的目標。

本節列出一些可能提供捐助的機構名單。雖然本文並不確指應該捐給哪個機構，但這份名單可作為“思考 的材料”。許多機構在美國均登記為慈善機構，因此在美國向這些機構捐款就可用于減稅。

下列組織的活動與用于 Linux 的軟件發展直接有關：

  • Linux International Project Sponsorship Fund
  • Debian/Software In the Public Interest
  • Free Software Foundation
  • The XFree86 Project

捐錢給這些組織就等於直接支持發展可自由散佈的用于 Linux 的軟件。以經濟效益而言， 這種捐助幾乎肯定比任何其他種類的捐款都更能對整個 Linux 事業產生更大的影響。

此外還有一些與 Linux 比較間接但也值得提供捐助的組織，例如：

  • League for Programming Freedom

    這不是一個與Linux直接有關的組織；它們進行與軟件開發人員有關的一般推廣工作。 參加這個組織的工作幾乎就等於參加一個“政治遊說團”。

    這多少有些“偏向美國”的味道，但無論如何卻具有國際影響，因為在與計算機有關的事務上，通常 國際社會都跟著美國跑，正如同美國跟著世界各國走一樣。

Linux 運動

我在這篇 HOWTO 堣@直引用我稱為 Linux 運動的東西。事實上，除了 稱它為一種運動之外，的確沒有一種更好的方法來描述目前世界上使用 Linux 的現 象。它沒有行政體制，但卻有妥善的組織；它不是一個合組的公司，但卻對全世 界的商業極為重要。一個地區性的 LUG 支持國際 Linux 運動的最好方法莫過於將當地的Linux組 織辦得生動活潑。Linux 是世界各國的使用者一起發展的，這 在/usr/src/linux/MAINTAINERS裡顯而易見。但Linux也在世界各國被使用。 使用 Linux 的人數日益增多，這正是 Linux 一直成功的關鍵。這也正是區域性的 LUG 能夠發揮 無可估量的重要作用的所在。

Linux 運動在世界各地風起雲湧，不僅僅是因為 Linux 的計算威力強大，價格低廉，而且還有發揮的空間。 最重要的是這套操作系統完全免費，不受任何人的控制。每次只要有人、有團體、有公司企業或組織機構 有機會了解 Linux 的特性和特點，Linux 運動的力量就得到加強，隊伍就更為壯大。地區性的LUG就能在這方面發揮作用。

4.4 Linux 的社交活動

地區性的 LUG 的最後一項工作是社交活動。這是最難討論的一項工作，因為一個 LUG 到底應該提倡多少 社交活動或投入多寡都很難有所定論。地區性的 LUG 不參與其他活動多少是件奇怪的事，但也的確有 LUG 并不以社交活動為其重要考慮。

然而三兩個 Linux 使用者混在一起時，比較心得、展示技術和有時喝兩罐啤酒幾乎是必然的事。Linus Tovalds 一向認為 Linux 的一個歷久不衰的特點：好玩。對玩家、內核發展人員和 Linux 使用者而言，沒有比下載一個 新的內核、新編一個老版內核、改編“視窗管理員”或修改一些程序源碼更叫人感到刺激的了。就是因為好玩， 許多 Linux 玩家聚在一起，也就是這種好玩，促使許多 LUG 自然地進行一些社交活動。

“社交活動”主要是指交流經驗、增進友誼和相互欽慕敬佩。不過，社會學家對此還有一層社會化 的意義。在任何一個運動、機構或民間社團中，都需要有一種過程或形式，以 Linux 的用語而言，使新手 變成玩家。換言之，社會化使你從“他們的一員”變成“我們的一伙”。

這個過程在美國部隊或許多其他國家的部隊中均稱為新兵訓練或基本集訓。通過這項訓練，一個 老百姓就變成了一名士兵。Linux 運動也有類似的需要。一個 Linux 新手應該學習作為一名 Linux 使用者的 意義何在、在世界性的大範圍中可以完成何種工作、Linux 運動的特別用語、所需的特別素質和擁有的特別 機會。這項工作可以簡單到在一個社區集會中指出“Linux”這個字如何發音，也可以困難到 Linux 使用者 應如何提倡推廣 Linux，或更重要的應如何克制不提倡推廣 Linux。

Linux 的社會化和“實際社會”中的社會化不同，可以通過電子郵件和 Usenet 的方式完成，儘管後者 的功效時常受到交往不足的使用者的質疑。我認為社會化和社交活動都最好在有相互接觸的地方進行， 而不是透過電子郵件或 Usenet。
