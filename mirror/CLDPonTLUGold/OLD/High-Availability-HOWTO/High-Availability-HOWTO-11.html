11. 未來的延伸部份

一些延伸部份列於未來的希望清單中：

  • 透過專屬的點對點高速 IP 網路作資料鏡射。 這考慮到橫跨任意距離通透的資料複製（雖然可能導致令人感興趣的財務經驗）。 因此，站台的災難回復是有可能的。 關鍵是一個位於一般的磁碟分割區之上並和位於遠端節點相應物交談的虛擬裝置驅動程式。 應用程式和檔案系統將位於虛擬裝置上以替代直接位於磁碟區塊裝置。 虛擬裝置驅動程式將捕捉所有對讀取與寫入的嘗試，以滿足本地端讀取並寫入到遠端及本地端。 關於這的更詳細概念將在往後發展但是對於那些進一步思考的人而言， 關於 IBM 的 High Availability Geographic Cluster （ HAGEO ） 如何運作就已經相當多了。 然而相對於 HAGEO ，將有個加密層級，例如透過 IPSEC 或 SKIP.
  • 發展或實作可被應用程式連結和可藉由某種方式讓應用程式控制叢集的叢集 API 。 現有的叢集 API 有 IBM 的 Phoenix （目前設計來處理多達 2048 個節點）或微軟在 Windows NT 上的 Wolfpack 。
  • 同步磁碟存取意指所有參與的節點都有讀取和寫入磁碟的存取權限。 只要還沒有智慧型的和有效率的鎖定機制，對檔案系統而言這將不會成為可能。 Oracle Parallel Server 是目前唯一使用這項特徵的商業應用程式。 我不知道 SCO 版本的 Oracle 資料庫伺服器（以 iBCS2 模擬器執行）是否支援同步存取。 如果沒有，請略過這點。 同步存取包括分散式鎖定管理程式的發展。
  • 以特殊的裝置驅動程式支援 "host-based" RAID 盒。
