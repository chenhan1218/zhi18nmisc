8.1 我要怎麼學得更多呢 ?

有一本 Linux Documentation Project 出版，由 Olaf Kirch所寫，叫 Linux Network Administrator's Guide的書是一本對想要設立及維護與 TCP/IP 和 Internet 相關的任何人都必備書。

ftp://sunsite.unc.edu/pub/Linux/docs/linux-doc-project/network-guide/nag-1.0.ascii.tar.gz

每一個軟體套件埵魊H著的文件通常會給你所需要的詳細資訊，雖然並不盡然是全面的。 而 man 頁是第一個必須看的文件。試試像： 

man pppd

你也可以在 /usr/doc 目錄樹堶惕鋮鴗@些 相關程式的文件，雖然它們並不是總是很結構化地呈現的。

下列的 HOWTO 都有很高的相關性：

  • Installation-HOWTO 有基本的資訊。
  • NET-2-HOWTO 有對安裝及設立網路的完整介紹。 如果你使用標準的 Linux 供應商版的套件(像 Slackware, Red Hat, Debian) 的話，大部份這些已被做好了。但它有很多關於設立以及故障排除的章節都非常有用哦！
  • Mail-HOWTO 解釋如何建構許多不同的工具。 再次地，如果你安裝了標準的 Linux 供應商版本的套件，大部份的工作已被做好了。
  • News-HOWTO 是為了設立一個 (一般的) news spool的文件。
  • Tiny-News 目前只包含一個蒐集新聞群組的替代程式。
  • PPP-HOWTO 埵雩埴荋y述當你在安裝 PPP 連接時可能會發生的問題。
  • Serial-HOWTO 有所有你在設立序列埠時所需知道的事。
  • Mail-Queue 告訴你如何來建立 sendmail 成為一面將 外部的郵件列隊排排站，一面將本地端郵件送出去的方法。

Red Hat 有一個關於 PPP 的郵遞列表，想加入它們只需寫信到

redhat-ppp-list-request 在主題上寫上 

subscribe
即可
