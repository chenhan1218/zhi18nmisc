2. SGML 相關問題

2.1 什麼是 SGML? 為什麼 Linux HOWTOs 要用 SGML?

答: SGML 是 Standard Generalized Markup Language 的簡稱，它允許你指定文件的結構 --- 也就是，用何種事物來標示文件。 SGML 使用"文件形態定義"(Document Type Definition, DTD)來指定文件的結構。而 Linux HOWTOs 採用一種叫做 linuxdoc 的 DTD 做為文件的結構。 詳細的說明，請參考 SGMLtools User's Guide 中譯版。

採用 SGML 做為 Linux HOWTOs 的寫作格式，是因為它可以轉換成為多種的輸出格式，包括了純文字，HTML，LaTeX, PostScript 等格式。最重要的，它為所有的 HOWTOs 文件提供了一致的外觀，便於管理與維護。請參考 這裡的說明。

CLDP 建議所有中譯的 SGML 文件開頭加上譯者及翻譯日期，如: 

<author>
作者: John <tt><htmlurl url="mailto:john@linux.org"
           name="john@linux.org"></tt><newline>
譯者: 張三 <tt><htmlurl url="mailto:chang3@linux.org.tw"
           name="chang3@linux.org.tw"></tt>

<date>v1.0, 1 January 1998
翻譯日期: 28 July 1998

2.2 我不了解 SGML, 我可以翻譯 Linux HOWTOs 嗎?

答: 是的! 當然可以。您不需要是 SGML 的專家才能做這件事。事實上，由於您也不是從頭開始寫一份文件，您甚至也不需知道共有那些 SGML 的標籤(tag). 您所要做的，就是先看一遍 SGMLtools User's Guide, 認識什麼是 SGML 的標籤。 然後拿起您要翻譯的 SGML 檔案，翻譯其中的英文句子，而保留所有的 SGML 標籤即可。

簡單的說，SGML 標籤就是由 < 及 > 所包含的英文字組成的。 另外一種特殊符號，是由 & 開頭而以 ; 結尾的英文單字。 因此，千萬不要將這個標籤 <author> 翻譯為 <作者>, 或將 <date> 翻譯為 <日期>. 還有包含在 <!-- 及 --> 之間的文字代表註解，可以不必翻譯。

另一需要注意的地方是交互參考的標籤 label 及 ref. 其中的 'id=' 內的字元只是參考用的符號，不需翻譯。 而 'name=' 內的文字為說明用，可以翻譯。例如: 

<ref id="reference6" name="Reference 6">
可譯成 
<ref id="reference6" name="參考資料六">

2.3 我翻譯的文件一定要採用 SGML 格式嗎?

答: 如果您要翻譯的是 Linux HOWTO, 那麼答案是 "對的!" 您一定要採用 SGML 格式。 因為所有的 Linux HOWTOs 都已採用 SGML 格式，原因也 如前所述。 而且所有 CLDP 的文件轉換工作都已自動化。如果您寄給我非 SGML 格式的文件，我無法處理，只好退回給您。

如果您要翻譯的是 mini-HOWTO, 那麼也可採用 HTML 或純文字格式(並非所有的 mini-HOWTOs 都已採用 SGML 格式)。 但若您要翻譯的 mini-HOWTO 也是採用 SGML 格式寫作，建議您還是也用 SGML 較適當。

請不要害怕。用 SGML 格式並沒有想像中的困難。請參考前面的解說。

2.4 如何轉換我的中文 SGML 文件?

答: SGMLtools 尚不能處理遠東地區的雙字元語系。因此除了安裝最新版本的 SGMLtools, 您還需要安裝 SGMLtools 中文套件 zh-sgmltools. 請在下面地點取得:

ftp://ftp.linux.org.tw/pub/CLDP/src/
zh-sgmltools-1.1.tar.gz
zh-sgmltools-1.1-x.i386.rpm
zh-sgmltools-1.1-x.src.rpm

安裝很簡單。解開後 

make
make install

詳細的使用方法，請參考 SGMLtools 中文套件使用說明。

另外如果您手邊也有 FreeBSD 2.2.x 或 3.0 的機器，有一個 FreeBSD package 'sgmlformat' 也可處理中文的 SGML 檔案。請依序安裝下面的 packages:

  • perl-5.00404
  • iso8879-1986
  • linuxdoc-1.0
  • docbook-3.0
  • jade-1.0.1
  • sgmlformat-1.6
(以上的 packages 皆可在 freebsd.csie.nctu.edu.tw 找到。)

然後將 /usr/bin 目錄下的 sgmlfmt, sgmls, instant 改名或刪除，並將 /usr/share/sgml 整個目錄改名或刪除，這樣就可以用新的 sgmlformat 了．

再來修改您的文件，將 DTD 的定義 

<!doctype linuxdoc system>
改為 
<!DOCTYPE linuxdoc PUBLIC "-//FreeBSD//DTD linuxdoc//EN" []>

要轉換為 HTML 格式，用 

sgmlfmt -e big5 -f html Foo-HOWTO.sgml

要轉換為純文字格式，用 

sgmlfmt -e big5 -f latin1 Foo-HOWTO.sgml

但目前 sgmlformat 仍無法轉換 LaTeX, dvi 及 PostScript 格式。
