1. 簡介

什麼是 Linux?

Linux 是一個 POSIX 作業系統的獨立實作，有著 SYSV 及 BSD 擴充，是從無到有整個寫出來的(這表示它看起來及運作上就像 Unix)． 它裡面沒有專利獨佔的程式碼．Linux 在 GNU Public License 下可自由地散佈．

它可在 IBM PC 相容機器，有著 ISA 或 EISA(包含了 local bus 的變種 VLB 及 PCI)及 386 或更高級的微處理器上運作． 除此之外，它也能在具有 MMU 的某些 Amiga 及 Atari 電腦上執行． 這是說具有外部 MMU 的 68020, 68030, 68040, 或 68060． Digital Alpha 的支援現已經很穩定．Red Hat 及 Craftworks 有 Alpha 版的 Linux 套件． Sparc 的支援現也很穩定．Red Hat 有 Sparc 版的 Linux 套件． PowerPC 的支援現在發展多平台版，包括了 Nubus 及 PCI Macintosh, Motorola Powerstack, IBM 830 和 850, 以及其它平台． ARM, StrongARM 及 MIPS 的支援現已完成許多階段，但仍不實用(hold your breath)．請見 comp.os.linux.announce．

這些平台支援的更多技術資訊請參見 Linux INFO-SHEET，確實的硬體需求參見 Hardware Compatibility HOWTO．

Linux 的核心是由 Linus Torvalds < Linus.Torvalds@.Helsinki.FI> 以及其它志願者所撰寫． 大部份在 Linux 上執行的程式是一般 Unix 的免費軟體，許多是由 GNU 計畫來的．

The Linux INFO-SHEET

上面有 Linux 更多的技術資訊．包括不同平台上的資訊指引，特徵列表，如何取得 Linux，及更多的資訊．

The Linux HOWTOs

有點像 FAQ，然而不是回答常見的問題，而是解釋如何做一般的工作，像是整理一份 Linux 的發行版本，建立 Linux 的列印服務，建立基本的 UUCP 餵送等等． 所有 HOWTOs 的最新版本可見 http://sunsite.unc.edu/LDP/HOWTO/HOWTO-INDEX.html 或 ftp://sunsite.unc.edu/pub/Linux/docs/HOWTO/． 其它具有即時拷貝的 HOWTOs 文件站台有 ftp.cc.gatech.edu 及 tsx-11.mit.edu．

除此之外，還有一些短的，任意格式的文件叫 "mini-HOWTOs"． 這些文件涵蓋了較特殊的主題，像是 BogoMIPS 或 Color-ls． 它們可從 ftp://sunsite.unc.edu/pub/Linux/docs/HOWTO/mini/ 及 http://sunsite.unc.edu/LDP/HOWTO/HOWTO-INDEX.html 取得．

Linux 新聞群組

有許多給 Linux 的專門新聞群組． 如果你使用 Linux，經常注意 comp.os.linux.announce 是個好主意． comp.os.linux.announce 現由 Lars Wirzenius 所管理． 要在此討論區上陳述意見，寫信到 linux-announce@news.ornl.gov． 你也可以將 comp.os.linux.announce 的問題直接向 Lars Wirzenius 陳述，地址是 wirzeniu@kruuna.helsinki.fi．

comp.os.linux.announce 是一個受管制的新聞群組，用來發佈關於 Linux 的消息(新程式，錯誤修正等等)．

comp.os.linux.answers 是一個受管制的新聞群組，用來發表 Linux FAQ, HOWTO 文件及其它的文件用的．

comp.os.linux.setup 是一個未管制的新聞群組，用來討論建立 Linux 系統所遭遇的問題．

comp.os.linux.admin 是一個未管制的新聞群組，用來討論 Linux 系統的管理．

comp.os.linux.development.system 是一個未管制的新聞群組，專門用來討論 Linux 的核心發展． 唯一可以在那裡討論的應用程式發展問題是與核心密切相關的． 所有其它的發展問題多半是一般的 Unix 發展問題而應被放到 comp.unix group，除非它們是與 Linux 有關的應用程式問題，這種情形的話問題應放到 comp.os.linux.development.apps．

comp.os.linux.development.apps 是一個未管制的新聞群組，專門用來討論 Linux 相關的應用程式發展． 它不是用來討論如何取得 Linux 的應用程式的，也不是用來給那些想看看 Linux 應用程式的人的討論場所．

comp.os.linux.hardware 是給 Linux 相關的硬體問題用的．

comp.os.linux.networking 是給 Linux 相關的網路發展及建立的問題．

comp.os.linux.x 是給與 Linux 有關的 X Windows 的問題．

comp.os.linux.misc 是一個未管制的新聞群組，用來給不屬於任何其它方面的 Linux 討論．

一般來說，不要在 Linux 新聞群組上重複發表(crosspost)文章． 唯一適合重複發表的是偶而發佈在一未管制的群組及 comp.os.linux.announce 的文章． 將 comp.os.linux 分割為許多群組的主要原因是為了降低每一群組的文章流量． 不遵守這項規則的話將會遭致他人毫不留情的抨擊…

其它的新聞群組

不要假設你所有的問題都適合於 Linux 新聞群組只因為你在跑 Linux． 你的問題其實是關於任何 Unix 或類 Unix 的 shell 程式寫作嗎? 那麼試著到 comp.unix.shell 上去問． 它是關於 GNU Emacs 的嗎? 那麼試著到 gnu.emacs.help 去問． 還有，如果你不知還有那些其它的群組可問，只是認為應該是這裡了，用有禮的態度試問是否還有更適合你問題的討論群組． 至少這些群組 comp.unix.{questions,shell,programming,bsd,admin}，以及 comp.windows.x.i386unix 應該是對 Linux 使用者有幫助的．
