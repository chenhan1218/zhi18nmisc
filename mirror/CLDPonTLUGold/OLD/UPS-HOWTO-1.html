1. 簡介

本 HOWTO 文件包含了如何連接 UPS 到執行 Linux 的電腦上。 其想法是以適當的方式將二者相連接，使得 Linux 可以在電源斷電以及 UPS 的電池耗光前安全地關機。

其中還包括了一些可幫助您建立上述連線的軟體，以及詳述該連線的運作方式。 假如您能找到一個已設定好可以使用您的 UPS 的軟體， 則後者的說明對您來說通常是不需要的。否則的話，您必須仔細地閱讀它。

在相當的程度上本文件已比起我在三年前所撰寫的原文多出了很多內容。 所有的基本資料都可以在 SysVinit 中的 powerd man page 塈鋮魽C 儘管在三年前有人可能無法在一般的 Linux 套件中找到這篇 man page ， 但我不相信今後還會再發生這樣的情況。

除此之外，當我第一次撰寫這篇 Howto ，除了 powerd.c 以外並沒有其他的軟體可以做為 Linux/UPS 之間的溝通與控制。 而今日我們則可以在 Sunsite's UPS directory 下找到一大堆 UPS 控制軟體。

然而，受到 RedHat 即將重新印製 Dr. Linux 的激勵，我也開始重寫這篇 UPS Hotwo 。 為什麼呢? 嗯 ....

  • 我想，額外的一般性概觀將能有助於了解如何將 Linux 系統連上 UPS ， 即使它只是另一份內容相同的文件。
  • 這篇 HOWTO 算是各種特殊 UPS 資料的儲藏庫 --- 在這兒有許多尚未被整合入一般軟體套件中的 UPS 資訊。
  • 這篇 HOWTO 包含了一些在其他文件中找不到的額外詳細資料。
  • 一些可以在 Sunsite's UPS directory 找得到的 UPS 軟體似乎說明文件都少得可憐。 您也許需要先閱謮本文以了解如何使用他們。
  • 現在這件事似乎已有了自己的生命了。當一篇 HOWTO 文件因而誕生時事實就更加明顯了。另一個比較不明顯的事實是當它必須上床睡覺時。 (譯註: 這一句笑話我看不懂啦! 唉! 翻得好爛 ....)

1.2 請特別注意

我真的無法保證這些東東對您會有效。連結 UPS 到電腦上是一件相當有技巧的工作。 其中之一或二者都有可能會燒掉、爆炸、著火、或者是引發第三次世界大戰。 除此之外，我只有對 Advice 1200 A UPS，以及 5kva Best Ferrups 有直接的經驗， 而且我也不需要自己做一條訊號線。所以，請注意! 儘可能地收集您的 UPS 的所有資訊。 請先考慮清楚，不要隨便相信任何您在本文中或其他地方所讀到的內容。

另一方面，我正盡可能地收集所有使我的 UPS 可以工作的方法， 其中有很多不是來自製造商的資訊，所幸也沒有燒掉任何東西，所以這些都是可能的。

1.3 其他的文件

本文並沒有包含所有 UPS 的一般特性與能力。如您需要這類型的資訊，你可以看看 The UPS FAQ。 這篇文件同時也可以在 ftp://rtfm.mit.edu/pub/usenet-by-hierarchy/comp/answers/UPS-faq 中找到。它是由 Nick Christenson ( npc@minotaur.jpl.nasa.gov) 所維護的，但看起來好像自從 1995 年以後就沒有再重寫了。如果您要 email 給他， 他希望您可以在信件的 Subject 那一行放入 UPS 或 UPS FAQ 或其他相關的詞句。

有許許多多的 UPS 製造商在網路中萌芽。 其中有些確實在他們的網頁中提供了有用的資訊。您可以在 The UPS Directory 中找到這些 UPS 製造商的網頁簡表。該網站也包含了 UPS FAQ 這篇文件。
