开放心胸, 开放原始码 (Open Mind, Open Source)

by Esther Dyson

当微软 (Microsoft) 的事业在法庭受到创击时, 它的软体发展模式和商务运作 却在家家户户受到创击. 这和 IBM 二十年前遇到的大问题极为相像, 反而比较 不像是反托拉斯法庭上的问题. Microsoft 也许应该在开放法庭中更加注意 开放原始码 (Open Source).

开放原始码 (以下会作长篇的解说) 基本上是由一群先前未协调, 但却一起 合作的程式师来发展软体, 他们使用可自由散布的原始码, 以及网路上的通讯 设施.

当然, 历史一再重演, 但从未一模一样. 我们怀疑 Microsoft 是否将会如从前 的 IBM 割让商场给竞争对手. 比较有可能的是, 就如同 Microsoft 对网际网路 一样, 它将认知到开放原始码对它商务模式的威胁, 并将了解如何去拥抱和开发 开放原始码. (也许对 Microsoft 而言, 开放原始码只是网际网路的威胁之一, 而我们在处於这场战争的中线.)

就开放原始码的案例来说, Microsoft 也许是幸运的, 它的万圣节文件吸引了 人们的注意, 但好在未变成大新闻. 这是一份 Microsoft 公司内部的备忘录, 不知怎麽流传到全球资讯网 (http://www.opensource.org/halloween1.html). 它是一篇关於软体历史的好文章, 是一份展示出商业和软体发展模式部份基本 问题的基础文件, 请和本文一起参照.

在这份文件里头, Microsoft 的员工 Vinod Valloppillil 清楚地指出: 对 Microsoft 的威胁不独来自 Linux, 而是广泛地来自开放原始码的观点. 就如 开放原始码的先锋 Eric Raymond 对这份文件做的注释中提到的一样, Microsoft 所损失的不只是程式除错可快速地并行处理, 也包括了竞争的创意, 这些创意在 市场中被其他人决定采用, 加强或忽略, 因而存活或逝去. 有趣的是, 这是一个 资讯的市场, 不会被单独一方的价格所阻碍, 也不会被其他人以量大或是行销所 阻碍.

(譬如说来自 BIND 作者 Paul Vixie 的新特色也许会比来自 Juan 或 Alice 的特色受到更多的试验) Microsoft 的观点一直问著: 领导人在那里? 而 开发原始码的观点也持续回答著: 领导人就是我们一致认为领先的人.

目前 Microsoft 的方法明显的是建立一家公司最好的方法; Bill Gates 的 股价高居不下, 但它也许不是发展商品软体最好的方法.

而这正是问题的关键: 一个赚钱的主要方法就是将市场导向到实用的商品 -- 卖一些能额外收费的东西, 当然这必须符合你商品的成本. 当其他人进入这个 市场时, 领先者享受暴利, 而後来的竞争者只能以爬行的速度在背後追赶著. 但在此时, 这个市场已转变成实用商品的市场. 到目前为止, 开放原始码的世界 仍缺乏企业导向的发展工具, 基础软体, 和支援. 但是当注意力和资金挹注至 开放原始码的世界时, 这个情形将会改变.

这也是目前 Microsoft 在系统软体所面临的挑战 -- 也许先前被认为是应用 (application) 的全球资讯网 (Web), 现在增加了许多使用者的工具, 以用来处理 这个具有丰富资讯的环境.

在万圣节文件中提到, 对抗开放原始码软体的方法之一就是制定专属协定 (也被称做整合). 但广泛地就 Microsoft, 或特别对 Bill Gates 而言, 他们都只有一套做法. 所以真正的问题是: Microsoft 会如何应用开放原始码的商业模式呢? 也许 它可以扩展顾问和客服业务 -- 一个稳定的收入来源, 但这项收入和雇用的员工 人数成正比, 获利的程度比不上贩卖软体的复本. 它也可以依法或被迫分裂成 数个不同业务内容的公司集团 (也许以它曾拥有的专属程式为基础, 在上面建立 开放原始码的软体). 这些单位 (译注: 分裂之後的公司) 也许再也无法达到 目前 Microsoft 所享有的经济规模和产能, 但那样的时代已经结束.

Microsoft 对真实情况显示出它单一的认知, 以它累积的财富和智库, 比起其他 公司, 它如果愿意放弃过去, 那麽在将来它会更容易成功.

开放原始码革命 (The Open-Source Revolution)

By Tim O'Reilly
(贡献这篇文章的人还有: Mark Jacobsen, Stig HackV□, Mark Stone 和 Dale Dougherty)

自由软体转为开放原始码 (Free software becomes open source)

就像电视影集 "X 档案" 中的剧情一样, 发生了一连串悬疑的事件:

一位在芬兰的大学生写了一个作业系统的核心, 并且公开发表它. 在六年内, Linux 的使用者膨胀到超过七百万人, 同时也是成长最快的伺服器 作业系统. (译注: 指 Linus Torvalds)

在宿舍里, 两位大学生使用可自由取得的的软体, 不只建立了新的商务,也建立了 数百万美元的公司. (译注: 指 Yahoo 的杨致远和 David Filo)

世界上最重要的计算机标准组织由参加它通信论坛 (mailing list) 的人们 所组成, 他们只有在一年三次的会议上才会出现. (译注: 指 IETF -- 网际网路标准制定委员会)

一些奇异的事正在我们面前进行著. 什麽在进行著? 又为什麽重要, 这就是 "科技简讯" (Release 1.0) 这期的主题.

一年之前, 如果你曾问过大公司资讯主管关於自由软体 (free software) 的事, 他会告诉你他们公司不使用, 他也许也会对你说原因是没有人支援, 不够强健, 未达商业软体的品质.

一下子之间, 在一片主流的声浪中, 这样的认知已经改变.

一月 (译注: 1998 年) 的时候, Netscape 决定释放旗舰级产品 "通讯家族" (Communicator) 的 原始程式码, 让人自由取得, 因而震撼了工业界. 虽然仍有官方版的产品, 但任何想制造和贩卖同类产品的人, 都可以自由利用该原始程式码作为基础. Netscape 这个决定是受到 Eric Raymond "奠基" 论文的鼓励 -- 教堂与市集 ( www.tuxedo.org/~esr/writings/cathedral-bazaar/中译), 该文中讨论 Linux 在技术上和市场上成功的关键, 在於以自由, 可重散布的原始码为基础 的发展方法. Raymond 宣扬开放原始码软体不只比独占码软体强健, 而且有较好 的支援及创新.

四月的时候, O'Reilly 公司 (本文作者所有) 举办了一场高峰会议, 邀集了自由 软体发展者中的领导人, 一起讨论 Raymond 论文的蕴涵, 以及努力提高自由软体 的气势. 该会议关键性的结果是这些发展者们同意采用 Raymond 的名词 -- 开放原始码软体 (open-source software), 以抛下和 "自由 (免费) 软体" 相关 的包袱. (虽然 free 有两个意义 -- 自由软体创始人 Richard Stallman 这麽说: 想成 "free speech" (言论自由) 而非 "free beer" (免费啤酒) -- 然而人们 已习於把自由软体当成非商业, 而不是非独占). 在相当快的後继反应, Corel, Informix 和 Oracle 宣告他们已经或在不久的将来要把他们的产品 移植到 Linux 上. 七月的时候, IBM 宣告他们加人 Apache 小组, 让 Apache 资讯网伺服器成为他们 WebSphere 这项产品的核心. IBM 也投入一个小组的 程式师, 协助将 Apache 移植到 NT 平台上.

九月的时候, Intel, Netscape, 名列前茅的投资公司 Greylock 和 Benchmark Partners 投资 Linux 的发行商 Red Hat 软体公司.

在此时, 人们才开始注意到原来有这麽多的开放原始码软体构成了网际网路的 基础: 网域名称服务 (DNS) 跑 BIND (the Berkeley Internet Name Daemon); sendmail 几乎触及了网路上每一封电子邮件, 像 Perl, Tcl 和 Python 这些 程式语言, 被 Yahoo!, C|NET, 和 Amazon.com 这些网站大量采用.

一下子开放原始码成了下一件大事.

但什麽是开放原始码呢? 公司该如何从中获利呢?

网际网路上原生的软体发展方法论 (The native development methodology of the Internet)

在早期, UNIX 和 Internet 的程式师们经常分享他们程式的原始码. 例如早先 AT&T 被禁止进入计算机工业, 所以 Ken Thompson 和 Dennis Ritchie 以象徵 性的收费, 将 UNIX 的原始码由贝尔实验室中送出. 数以百计, 然後数以千计的 程式师, 用在他们的工作上, 贡献个人的工具程式, 移植到别的平台, 总而言之, 就是把这些软体弄到符合他们个别的需求.

在这些需求中, 最迫切需要的就是找到一个更简单, 更快速的方法, 能让大家分享 工作成果.

於是网路出现, 方便了沟通和分享程式码, 这个网路原先以 UUCP (Unix-to-Unix Copy) 拨接协定为基础, 任何人都可以加入, 你所需要的 只是一个站台, 而对方又愿意和你连线.

当时, 大学和研究所藉由美国政府的辅助, 彼此以专属的高速线路连结, 他们采用 另一个网路协定 TCP/IP (Transmission Control Protocol/Internet Protocol). UUCPnet 和以 TCP/IP 为基础的 ARPAnet 共享一个巨大可靠的讯息传递系统, 称为 Usenet. Usenet 原本的目的是为了散布计算机程式的原始码和相关的资讯 与讨论, 很快地, 它演变成一种社会媒体.

1991 年发生了两件事, 转变了计算机工业: Rick Adams, 是美国 Geological Survey 的一位系统管理员, 该单位拥有 Usenet 最大的讨论聚集站, 他认为 该是成立商业 Usenet 基础建设的时候了, 他设立了 UUNet 以提供商业 UUCP 和 TCP/IP 的网路服务.

Tim Berners-Lee, 是 CERN 位於 Geneva 高能物理实验室的程式师, 他发展出 一种散布资讯的超文字, 主-从式 (client-server) 系统, 称为全球资讯网 (World Wide Web).

就是从那时候起, 每个人才知道商业网际网路和全球资讯网的历史. 伊利诺大学 的一个团队在 Berners-Lee 的工作成果上建造了 Mosaic 浏览器, 接著 这个团队组成了 Netscape 公司, 然後 Microsoft 也信奉起网际网路, 商业网站 激增, 到了 1997 年, Microsoft 与 Netscape 之间的战争, 令人感觉到彷佛 网际网路是这两家公司的产品一般.

事实上, 网际网路是由各自独立的发展者所孕育, 他们建造工具, 以使人们进行 更多的合作, 而合作得更有效力.

以下是由 Raymond 的论文 "教堂与市集" 节选出来的片段, 它们表达出这个运动 的特色:

"最好的程式起自於作者个人要解决他每天的切身之痛, 然後因为这通常也是许多 人的痛处, 所以这个程式便开始散布".

"Linux 的世界从许多方面看来, 像是一个自由的市场或生态, 由一群个体所 组成, 这些个体以一种自发性的自我更正程序, 试著去发挥他最大的功用, 所 发挥出来的功用比起集中式的规画要来得更精巧, 更有效率".

"优秀的程式师知道要写程式, 伟大的程式师知道要改写 (和重覆利用) 程式. ... ...

他们认为人们争取最好的成绩并不是为了努力的过程, 而是为了最後的结果. 更何况由一个部份可行的解决方法开始总比什麽都没有容易得多".

"尽早, 经常发表新版本, 并且倾听使用者的意见. ... ...

把你的使用者视为协同发展人, 可以让你伤最少的脑筋, 但做到原始码的快速 改善, 程式的除错有效率. ... ...

以足够多的 "beta 版" 测试者和协同发展者做基础, 几乎程式中的每一个问题 都可以很快地找出来, 并且对某些人而言, 针对发现的问题的解决方法是 显而易见的. ... ...

或者用比较不那麽正式的说法: '足够多的人来看程式, 所有的错误都变得浅显'"

开放原始码专案数以千计, 其中每一个都有许多发展者一同运作, 以下是其中 比较重要的:

  • Linux. 以 Linus Torvalds 的核心为基础, Linux 的发行通常包含好几百个 其他的发放原始码套件. (其中大部份的套件也可以在其他平台上执行, 包括 商业版的 UNIX, Windows 和 Windows NT, 麦金塔, 以及其他等等.)

  • FreeBSD, OpenBSD 和其他 Berkely UNIX 的衍生者, 虽然 Linux 获得大众 的青睐, 但是 BSD UNIX 系统也有相当多的拥护者.

  • 程式师的工具. 自由软体基金会 (The Free Software Fundation) 的 GNU 专案产出许多高品质的程式师工具, 其中包括 gcc C 语言编译器, g++ C++ 语言 编译器, emacs 编辑器, gdb 程式除错器. 另外还有两个重要的工具, 虽然不在 GNU 专案之列, 但却是开放原始码文化不可或缺的, 一个是 Larry Wall 的 "patch" (修补) 工具, 它可以让发展者交换对原程式的小量修补, 而不用搬出 修改後程式全部的原始码, 另一个是 CVS, Concurrent Version System, (版本同步系统), 它可以让发展者维护多个原始码版本.

  • 程式语言. Larry Wall 的 perl 语言无疑地是开放原始码界的程式语言之王, 而 John Ousterhout 的 tcl, 和 Guido van Rossum 的 Python 也拥有许多 支持群众. (Larry Wall 现在是 O'Reilly 公司的员工.)

  • Apache. 如同先前提到的, Apache 小组主宰了资讯网伺服器的市场. 最近 根据 Netcraft 网站的调查 (www.netcraft.co.uk/survey) 显示: 在所有公开 的资讯网网站中, 使用 Apache 的占 53%, 之後是 Microsoft 的 IIS 占 23%, Netscape 占 7%. Apache 小组是在 1995 年由一群网站的管理者所组成, 他们共同分享彼此对原来 NCSA 资讯网伺服程式 (web server) 的修改, 因为 原作者 Rob McCool 离开了 NCSA, 进而 NCSA 放弃继续维护该程式. (NCSA 是美国国家超级运算应用中心 -- National Center for Supercomputing Applications --, 位於伊利诺大学, Mosaic 浏览器亦是由该中心所产出) Apache 资讯网伺服程式仍在持续发展, 主要由 12 位核心发展者组成的小组 负责维护, 再加上庞大而活跃的使用者族群.

  • Samba 由一个成员遍及世界各地的团队所发展, 该团队由在澳洲的 Andrew Tridgell 领导. Samba 可以让 UNIX 和 Linux 变成在 NT 和 Windows 95/98 网路上的档案及列印伺服器. 这招秘技可以让管理员把 Linux 加 到他们的网路中工作而不需要去管它. Silicon Graphics 最近雇用 了一位 Samba 的重要发展者, 就是因为他们发现 Samba 是一项很棒的工具, 可以帮助 SGI 伺服器的销售.

  • Sendmail 原来是被发展成 Berkeley UNIX 的一部分, sendmail 是网际网路 电子邮件伺服器的骨干.

但只看这些最著名的软体仍不足涵盖开放原始码软体. 当你轻掀软体的表层後, 就会发现处处都有开放原始码软体. 几乎每家厂商的 TCP/IP 堆叠 (包括 Microsoft 的) 都源自於 Berkeley UNIX 网路套件的一部分. 网际网路的 基础建设和标准的产生过程都是开放原始码运动最好的□例. 就目前最尖端的 技术而言, 大多数商业版的 XML 套件都是以 James Clark 所写的开放原始码 xml 语法分析器为基础, James Clark 是一位住在泰国独立工作的程式师, 他 也写过许多 GNU 的文字处理工具. (译注: 如 groff)

判断开放原始码社群的人数 (Sizing the Open-Source Community)

估计不同开放原始码社群的市场大小, 是一个挑战, 因为没有明确的研究先例, 为了进行计算, 我们采用一个大家接受的估计值, 即以七百万的 Linux 使用者, 做为开始的依据 (请参考 Red Hat 在 www.redhat.com 或 1998 年九月份 "软体杂志 [Software Magazine]" 的论文 "判断 Linux 市场的大小 [Sizing the Linux Market]").

然後我们调查在不同讨论群组 (news group) 张贴文章使用者人数, 他们都有各自 所属的开放原始码社群 (使用由 netscan.sscnet.ucla.edu 提供的分析工具 -- Netscan, 分析的时间由 1998 年 9 月 2 日到 1998 年 10 月 2 日). 由这些 讨论群组的参与人数, 我们以七百万个 Linux 使用者为基准, 用外推法估算 每一个社群的人数.

商业模式 (Business Models)

对 "科技简讯" (Release 1.0) 的读者而言, 以上所说可能都非常有趣, 但是一个大问题是: 你要如何靠开放原始码赚钱呢?

待会我们会讨论到, 许多公司靠开放原始码软体赚钱, 少部分的公司靠销售 开放原始码的产品或衍生产品赚钱. 我们将此分成三种基本模式:

建立品牌发行 (Branding and distribution)

以值得信赖的品牌和既有或新辟的销售管道, 发售最新版的软体, 是靠开放原始码 赚钱的一个方法. 结合值得信赖的品牌通常是为了提供保证, 保证售後安装短期内 技术支援, 以及针对可能遭遇问题的额外支援套件.

Red Hat 软体公司 (www.redhat.com) 是这个模式的佼佼者. 若要真的 了解 Red Hat 公司的商务, 你首先必须知道: 其实 Linux 作业系统本身集合了 许多开放原始码软体, 只是这些软体建构在 Linus Torvalds 的核心上. 当 Linux 越来越普及, 也就有了许多不同的发行版本, 每一种版本都各自整合了不同的 开放原始码套件, 包含了不同硬体装置的驱动程式.

在 1994 年, 一位卡耐基美隆大学资科系的毕业生 Marc Ewing 到 IBM 工作. 他利用空□的时间使用两台 Linux 工作站, 建造一套高阶的程式发展工具, 这两台工作站执行的 Linux 版本, 都是他做了相当的修补和修改而来. 当他发现自己无法在专案上达到预期的进度时, 他找到了原因, 原来他花了相当多 的时时间在维护 Linux 工作站. 之後, 他做了决定, 世界上并不真的缺一套程式 发展工具, 但却真的需要一个较好的 Linux 发行版本. 所以他开始进行 Red Hat Linux 计画, 把他个人在 Linux 上遇到的限制定位出来.

不久後, Red Hat 公司最大的客户, 一家小型免费连线软体发行商 ACC PC UNIX and Linux Catalog 的总经理 - Bob Young 向 Marc Ewing 表示, 他不想只卖这个产品而可以做更多事, 他提议结合彼此的努力, 把他在市场行销 的专业知识加注到他们的发展团队. 於是 Young 担任主席, 执行长, 以及首席 福音传播者, 只在最近总经理换成了 Matthew Szulik, 之前是 Relativity 软体公司的总经理. Red Hat 公司在出资者的支持下於 1997 年 8 月度过了 它第一个会计年度, 在第二个会计年度, 於 1998 年 9 月, Greylock, Benchmark Partners, Intel 和 Netscape 提高了它的非封闭的资产额. 到今天, Ret Hat 公司的年度获利超过美金一千万元.

Red Hat 制造的套装软体, 包括了 Linux 作业系统和工具程式集, 采用让人自由 散布的 GPL (Generic Public License -- 公用大众版权) 版权声明 (详见以下 方块文章). 初级版的 Red Hat Linux/Intel 售价 49.95 美元, 它可由线上订购 或由零售商处购得, 除此 Red Hat 也提供标准版和黄金版的支援套件.

Red Hat 公司是 GPL 的忠实拥护者. Young 指出: Red Hat 的成功, 部分是因为 它降低了修改软体以符合特别需求的交涉成本, 独占的版权会在此成为障碍, 因为它需要和原厂交涉和取得其同意.

Young 表示: "开放原始码让使用者能够真的掌握作业系统的技术, 这就像是 IBM 发表原来个人电脑的规格时, 所给予电脑购买者的权利一样, 消费者每次都能 掌控并加以选择. 这种掌控驱使系统的发展者造出更可信赖, 更稳定, 以及更 安全的系统, 而且比只提供机器码的独家系统更便宜."

Young 鼓励人们复制并贩卖 Red Hat 发行的 Linux, 事实上, 它可由 www.cheapbytes.com 下载, 或只要花 1.99 美元, 就可取得. 他把这当成是 广告费用而不是收益的损失. 这个策略的基础在於: 即使有众多的产品 (通常 由相类似的厂商提供) 可供选择, 但人们还是会选择有品牌的产品.

Young 非常相信 Esther Dyson (本刊的编辑) 的理论, 并对 Red Hat 的商务 策略有相当的影响, 他忆及 1993 年的一则文章: "她 (译注: Esther Dyson) 的 论文提到, 当技术的发展周期变短, 拥有技术本身的价值也跟著变少, 但这个 技术所打出品牌的价值却增多."

Young 最具煽动性的言论就属: 他的目标是 "要把作业系统市场的消费总额降低", 把他觉得价格被过度哄抬的产品市场合理化, 他并不想要落在 Microsof 数十亿的 业务之後, 他认为如果他能把作业系统市场的消费总额降低至 5 亿美元, 那麽 Red Hat Linux 会有很高的占有率.

Young 也相信他的业务是以服务和支援为主, 而非不断贩售同一套产品的复本. 他说: "事实上我们赠予技术是为了获得像大规模技术公司在服务上的收益. 迪吉多 (Digital Equipment Company) 由技术支援和服务获得的收益超过 他们卖出的电脑, 而它是一家每年营业额 150 亿美元的公司."

其他开放原始码的包装商, 如 Walnut Creek CD-ROM (胡桃湾光碟公司) 发行 售价 39.95 美元的 FreeBSD 作业系统, S.u.S.E. (欧洲最大的 Linux 发行商) 提供最新版的 Linux 作业系统, 一本参考书和一套管理工具程式, 售价 49.95 美元. 最重要的是这些公司都能回馈开放原始码社群. Red Hat 和 S.u.S.E. 不但开发 Linux 上的程式, 同时也将它贡献给社群. Walnut Creek 提供工作 给好几位 FreeBSD 的发展者以支持 FreeBSD 的发展.

在建立品牌发行的模式下, 事实上消费者是支付给三个项目: (1) 在光碟上的原始程式码; (2) 发行公司对上面的原始码是最新 (非永久), 最稳定的版本的保证; (3) 通常包含支持发行的赞助, 有时是额外的费用. 就某些方面看来, 消费者支付额外的费用给开放原始码, 就像为了环保而多付出一些 费用购买 "绿" (green) 的产品.

注入附加专属价值到开放原始码 (Addition of proprietary value to open source)

运用开放原始码赚钱的第二种商业模式, 是发展开放原始码的加值独家产品, 然後这些产品主要被卖到实用市场. 因为产品中包含独家的心血, 所以一般来说, 这些产品并未回馈给开放原始码社群.

Sendmail 公司 (www.sendmail.com) 是这类型公司中知名度最高的.

Eric Allman 早先在加州大学柏克莱分校参与发展 BSD 时, 负责撰写 sendmail 程式, 在许多开放原始码专案中, 这是一个 "解决切身之痛" 的典型例子, Allman 在柏克来的小组曾非常渴望和 Arpanet 连线, 连上之後, 其他的研发 人员也想分享, 而 Allman 认为写一个帮他们送信的程式, 要比帮他们在自己 机器上建帐号容易得多.

Allman 写的这支程式附在 1979 年的 Berkeley UNIX 中叫 delivermail. 之後有一个新的协定被发展用来传送电子邮件, 叫 SMTP (Simple Mail Transport Protocol), 於是 Allman 把 delivermail 改进成 sendmail. 这支程式很快地 成为 Arpnet 然後是 Internet 上主要的邮件传送代理者. (当你在 Internet 上 发送电子邮件时, 你电脑上的邮件客户端程式并不负责把信送到目的地, 事实上, 你的邮件会通过一连串的邮件伺服器, 这些伺服器会帮你找出到达收信者最直接 的路线.) 大约有 80% 的网际网路使用 sendmail 做为它们的邮件传送代理者, 因为邮件由发信者到送信者间会经过好几台机器, 基本上每一封送上网际网路的 电子邮件都会有一台在某处的 sendmail 伺服器在处理.

在 1997 年年终, Allman 决定要将 sendmail 商业化. 徵召 Integrated Systems 和 Sybase 的前副总裁 Greg Olson 成为他公司的执行长, Allman 很快地由出资 者把资本额提升到 100 万美元, 出资者包括 Sun 的开国元老 Bill Joy 和 Andy Bechtolsheim, John Funk (Infobeat 的执行长, 该公司名列最佳电子邮件 服务商之一, 也是世界上 sendmail 的最大使用者, 每天发出的电子邮件超过 六百万封), 以及出版商 Tim O'Reilly (本文作者). 1998 年年中, Olson 把 资本额提升到 600 万美元. Olson 开始谈及 ??VC??, 但他在 Silicon Valley Band (矽谷带) 投资公司所做的报告实在太精彩了, 反应十分良好, 所以他决定 拿下这笔资金并继续他带动 Sendmail 公司业务的工作.

Allman 决定走上商业化的原因具有前导的意义, 他说:"当 sendmail 发展到 这个时候, 我已经要花上所有的时间来维护它. 如果要让它更好, 我需要更多 的资源, 尤其是新的发展有更迫切的需要." 另一个原因是电子邮件 SMTP 标准 渐渐地被一些独家的格式侵害. Allman 觉得 sendmail 做为一个转送信件的 媒体, 遵循标准是 Internet 成功的一个重要原因. 如果没有更多的资源来支援 sendmail 主宰市场, 他担心不只 sendmail 会被其他相竞争的产品取代, 甚至 开放的电子邮件标准亦深受威胁. 如同许多开放原始码的商务, sendmail 结合 了理想与现实. Sendmail 公司计画生产商业版加值工具程式, 提供服务给非常 重视电子邮件的企业和 ISP (网际网路服务供应商), 同时也继续创新并保持 开放原始码的发展路线. Olson 指出假如 Sendmail 公司能吸引少部份的 ISP 和大公司信赖 sendmail, 那他就能带起可观的业务.

Allman 的务实, 混合商业模式是以传统的生产线方式为基础, 商业生产线提供 加强版给有需要的商业使用者, 例如每一分钱都要有效用, 有图形化使用者介面 及已测试过的软体套件, 以及完整的商业服务, 包含市场, 行销, 训练, 技术 支援及谘询. 而开放原始码生产线则提供给网际网路上的程式发展者, 及开放 原始码软体的使用者, 他们需要用来建立讯息基础系统, 而这个系统正是建立 於被证明过符合标准的技术之上.

Sendmail 这个混合的商业模式让不同的顾客都获得好处. 邮件伺服器的新功能 和新标准会以开放原始码的型式先出现, 这加速了 sendmail 的创新, 并且确保 这些加强处的品质, 因为这些程式码会经过全世界网路上专家的检视, 使得 sendmail 成为验证完整, 发展最佳的程式. 商业版 sendmail 供给开放原始码 版的 sendmail 发展资源, 并且也满足商业上客户的特别需求. 在这样的方式下, 除了以封闭的观点, 意图垄断网际网路上某段商业网路的纯粹独家厂商外, 每一个人都是赢家.

Sendmail 公司称自己的经营模式叫 "混合型", 它同时拥有商业和开放原始码的 市场, 但其中只有一个能带来直接的收益. Sendmail 公司在开放原始码社群 发表 sendmail 8.9 版, 是它第一次扮演法人团体的角色, 它想要表达一个强烈 的讯息, 就是成立一家藉 sendmail 获利的公司, 并非表示它放弃了开放原始码 社群. Sendmail Pro, 即 sendmail 第一个商业版, 预计在 1999 年年初发表, 并将於 12 月在波士顿举行的 Usenix LISA 会议上展示.

在温哥华的 ActiveState Tool 公司 (www.activestate.com) 是另一家 采行这种混合型商业模式的公司. 1995 年时, 该公司的创立者 Dick Hardt (後来经营 Hip 通讯公司) 和微软订下合约, 发展了最早移植到 Windows 平台 的 Perl (微软把它收录到 Windows NT 的 Resource kit). 因为在 Windows 的 社群中, 开放原始码并不是一个普遍的观念. Hardt 发表的这项产品, 他称作 "机器码的自由软体", 也就是说这个软体的执行档可以自由散布, 但是原始码是 Hip 公司的专利.

1997 年, Hardt 出售 Hip 公司发展网站的业务并成立新公司 ActiveState (由 O'Reilly 出版公司投资), 集中火力於发展 Windows 上的 Perl. Perl 已经是 UNIX 和全球资讯网站的主要语言, 它也逐渐成为 Windows 上重要的 自动化工具.

开放原始码授权书之简介 (A Brief Introduction to Open-Source Licensing)

在最佳的情况下, 开放原始码的授权书能够培养软体发展人和软体消费者间 合作, 共享和共生的关系. 许多暨有的开放原始码的授权书已经建立成功 的模式. 这里由最老到最新列出三个最重要的授权书.

BSD 式的授权书 (BSD-style licenses)

BSD 式的授权书 (这个名称源自於 Berkeley Standard Distribution UNIX) (译注: 根据 daemon book, BSD 应是 Berkeley Software Distribution 的缩写) 最为悠久和宽松. 它们授权给予衍生的私有软体 (即如传统的商业软体不提供 原始程式码), 至於衍生出的改变是否要回馈给原来的公用版本, 则由厂商自行 决定.

开放原始码社群中有些人, 对於 third parties (非主流厂商) 由公用的软体库获利, 但却没有贡献, 感到愤恨不平. (在经济学中, 称此为 "搭便车" 问题), 仅管没有任何的要求, 这样的授权仍引来许多志愿者彼此合作. 许多网际网路上的重要软体都是 BSD 式授权的软体, 如 BIND, Apache 和 sendmail.

GNU 通用公众授权书 (GNU general public license -- GPL)

GNU 的通用公众授权书由 Richard Stallman 在 1983 年所拟, 该授权实现了 GNU 专案 "反版权" (Copyleft) 的观念. "版权" (copyright) 阻断了复制和 衍生工作的可能性, 但 "反版权" 却允许无限制的复制和修改. 然而, "反版权" 的使用者必须担负起一些责任, 散布衍生的原始码, 必须免费, 且只能采用 "反版权" 式的授权.

GNU 专案的重点是 "free 的软体, free 这个字在这里是自由而不是免费". 你可以出售自由软体, 但必须同时附上原始码. GPL 很严格以致於很难和 采其他授权的软体共存, 假如你加强了一个 GPL 的软体, 那麽你加强的部份 也必须采用 GPL, 然而 GPL 有一个比较宽松的型式 -- LGPL, 专供函式库 采用.

GPL 擅长於防止独家的垄断行为, 独家垄断对 UNIX 市场带来莫大的伤害, 采用 GPL 的成功□例包括 Linux 核心程式 (kernel), GNU C 语言编译器, 和 Samba 档案伺服器.

Mozilla 公众授权书 (Mozilla public license -- MozPL or MPL)

MPL 是网景公司 (Netscape Communication) 通讯家族开放原始码版本 5 的 一部份, 它试图找出 BSD 式授权和 GPL 之间的均衡点. 在 MPL 之下, 允许个人自行发展衍生软体, 但是 MPL 所涵盖的原始码若有修改, 则必须在网际网路上公开, 让其他人可自由取得, 而 MPL 比较宽松的地方在於: 额外增加的原始码 (不是修改过的原始码) 如果构成一个比较大型的专案, 那麽这个额外增加的部份可以采用不同的授权, 而且也可以不公开.

开放原始码的授权方式, 正如同开放原始码的经济模式一样, 都在演进中.

如同 Sendmail 公司一样, ActiveState 第一步的行动就是要接触开放原始码 社群. 最先移植到 Win32 的 Perl 和 UNIX 上 标准的 Perl 有相当多的歧异. 在 Larry Wall, Gurusamy Sarathy (另一支移植到 Windows 上的 Perl 的 作者), 和一些其他的发展者的一同工作下, Hardt 将公司第一年的时间花在 OnePerl 上(以该通信论坛的名字命名, 许多工作是在这个讨论区上完成 的). 1998 年夏天发表的 Perl 5.005 含括了 OnePerl 的工作成果.

就在 Perl 5.005 发表之後, 才在市场上种下好产品的种子, ActiveState 公司回到它专利的业务, 它发展了一套加值的 Perl 产品, 供给 Windows 的市场, 包含了一支程式除错器, 由 Perl 产生 COM 物件的工具, Perl 的 变种 PerlScript (和微软的 Active Server Page 一同运作), 以及许多 其他的产品, 让使用者得到 Windows 特别功能的好处, 这些套件售价由 100 美元 到 395 美元.

Cygnus Solution 公司 (www.cygnus.com) 提供开放原始码技术的支援和 客户工程服务, 该公司发展增强的 GNU 工具, 以 GPL 的版权, 发表回馈给网路 上的社群, 虽然 Cygnus 直接出售和支援这些工具. 例如, 微控制器的公司支付 Cygnus 公司服务费用, 以使这些工具能用在他们 32 和 64 位元的微处理器上.

Cygnus 公司的销售量, 自 1992 年起, 每年成长率超过 65%, 过去的四季总共 获利超过两千五百万美元. 该公司在 1989 年时, 由 Michael Tiemann 和 John Gilmore (Sun 公司早期的员工, 也是一位自由软体界的领导者) 共同创立. 1997 年 2 月时, August Capital 和 Greylock Management 投资它超过六百万 美元. (一个走漏的风声: Esther Dyson 是该公司的投资人兼股东)

[译注: Cygnus 现已被 RedHat 买下]

在周遭赚钱 (Make your money on the side)

开放原始码新出现一种商务模式, 我们称之 "在周遭赚钱" 的策略. 在这种模式 之下, 公司的重点不在开放原始码或独家的加值软体, 而是藉客户对开放原始码 的信赖, 使公司更能接触和帮助客户来赚钱.

IBM 在 1998 年 6 月宣布它将以 Apache 附随 WebSphere 应用伺服器一起出货, 因为 Apache 变成了 IBM 套装软体的一部分, 所以 IBM 将提供商业上, 企业 水准的支援. 因为 Apache 是市场上信誉的领先者, IBM 希望因此为它的加值 Web 应用伺服器赢得信誉 (及销售率). 对 Apache 小组而言, 最大的收获是 IBM 发展团队的加入, 因为 IBM 将集中发展的资源, 以改善 Apache 移植到 NT 的版本, NT 版的 Apache 一向比不上广为人用的 UNIX 版.

网景 (Netscape) 在 1998 年 4 月释出它 "通讯家" (Comunicator) 的原始 程式码. 网景的策略是加速未来高品质版本的发展与散布, 以提供给商业上的 客户及个人. 这麽做, 等於在网景企业解决方案的商业市场, 顾问服务, 和 Netcenter 的业务撒下种子, 这些将会变成公司主要的获利来源.

Corel, Oracle 和 Informix 已经公开宣布他们正在进行 (或己经完成) 移植 他们的产品到 Linux 平台上. 藉此应该可以增加他们产品的潜在用户, 并让 这些产品更便宜. 像 Corel 的新产品 Netwinder 就以 Linux 为重心, 作为一个 平价的轻量伺服器 (thin-client server).

在开放原始码这场游戏中的间接赢家 (Hidden Winners in the Open-Source Game)

当大家集中注意力於致力开放原始码的公司时, 一些公司把开放原始码软体 当成跳板, 并且赚了大钱, 这里列出这些赢家:

UUNet. 如先前所提, Rick Adams 是开放原始码软体 B News (供 Usenet News 使用) 的发展者, 同时也是世界最大的 Usenet 站的管理员. 当 Usenet 在大量使用者 的压力下, 它的骨架已经负荷过重. Rick Adams 察觉到该是有商业服务的时候了, 於是他创立第一家 ISP, 之後这家公司赚进了数亿美金.

思科 (Cisco). 虽然 Cisco 在 IETF (网际网路工程特别小组) 中相当活跃, 但没人会把它误认 为开放原始码的公司. 该公司爆炸式的成长实拜网际网路扩张之赐, 假若没有 开放的标准, 没有创新的开放原始码软体, 或者进入网际网路的门槛没这麽低, Cisco 今天的规模也不会这麽大.

微软 (Microsoft). 没错, 也许这个案例看来有些牵强, 但只要我们想到: 整合网际网路正是 Windows 98 吸引人的新特色, 假使没有网际网路, Microsoft 很难以它自己为是 的创意赚到钱.

开放原始码和商业活动间的关系比我们所相信的更复杂 -- 也对这个产业更 基础 -- 稍後我们会在 "新文艺复中的科学" 一节中解释.

这些商业模式的成功或失败将会影响开放原始码运动的成长, 开放原始码的成长 若要跨越到发展者社群 (developer community) 之外, 它必须能被资讯系统的 主管所接受, 就像 Microsoft, Sun, 和 HP 的解决方案一样, 这只有当资讯 主管看到在开放原始码之後, 有获好评, 可依靠, 和可信赖的公司, 以及好用 的工具时, 才有可能发生.

这些公司仍在初期的阶段, 其中如 Red Hat 公司, 已经成立好几年了, 但目前 只发展商业, 企业为主的支援能力, 当投资者挹注资金给这些公司, 以帮助它们 培养支援能力, 以及投资者发现如何扩大这些公司的规模, 并以开放原始码的方式 赚钱时, 那麽采用开放原始码的人应该会急剧增加.

开放原始码的公司也许无法成长到和 Microsoft 一样的规模, 但却能在市场上 获得成功, 他们需要的只是盈收超过支出.

开放原始码: 多方进行软体发展 (Opensourcing: Distributing Your Software Development)

为了协调软体发展者, 开放原始码的发展方式显得特别重要, 许多这类的专案 并不是为了要卖钱, 而只是简单地为了解决各自遇到的商业问题, 有时候这些 解决方案只是私有的或只有单一公司获利, 但常常一间公司就只需要一些可以 运作的软体, 许多开放原始码专案以此方式开始, 而当发展者发现还有其他人 也在解同样问题时, 这个专案就开始流传到世界各地.

网际网路增加了不同机构间彼此合作的机会, 於是出现了多方进行软体发展这种 新模式, 发展者之间因此可以合作得更紧密, 因为开放而分享到更多. Eric Raymond 用 "把全世界当成(你)的智库" 这句话来形容这个情形.

让软体发展跨越单一的组织也许是开放原始码运动中最有意义的一面, 我们称 这种新发展模式叫 "开放原始码" (opensourcing), 它的含意是 "原始码流出" (outsourcing). 许多公司如果更积极地投入开放原始码, 那麽他们会成为 大赢家.

开放原始码能让小型软体发展团队壮大, 就最简单的情形来说, 它 能找到和你有相同问题的人, 并且把他们拉拢进来一同努力. 它使得 软体发展的成本分散到多个不同的组织, 并且增加了高手贡献的机会, 开放原始码的结果正是 -- 贡献者社群组成了一个工作共同体.

一个开放原始码的专案是如何起始的呢? Raymond 认为要建立一个发展程式的 社群, 必须由起始者提出 "合理的承诺", 刚开始的程式码 (也许很糟) 要能 展示出基本的想法或是未来的发展方向. "最好的程式起自於作者个人要解决 他每天的切身之痛, 然後因为这通常也是许多人的痛处, 所以这个程式便开始 流传".

开放原始码的发展方法是可行的, 大部份的公司需要发展软体但不想卖软体, 很少有公司自己下赌注去开发一个产品的软体部份.

发展软体的成本很高, 维护既有软体的成本一样很高, 软体若不能达到目标 就一无是处, 开放原始码则能增多对同一软体下注的人, 因此可以降低发展成本 并且增加发展软体所需的资源, 这也让软体的发展者和协助除错的使用者的人数 增加, 除此之外, 开放原始码的过程培育了能维护这个软体的社群.

开放原始码与全球资讯网 (Open Source and the Web)

建造下一代的资讯体产品 (Building the next generation of infoware products)

很多人都问我: Linux 是否真的有机会打败 Windows, 但是这个问题问错了, 开放原始码已经根本地改变运算游戏 (电脑界) 的规则, 它所带来的涟漪效应 正用力地把重力的中心移离 Miscrosoft.

为了说得更明白, 让我们由一个小故事开始.

最近我和一些没有电脑的朋友聊天, 他们正考虑要买一台, 因为他们想用来向 Amazon 买书和 CD, 他们并不是想上网际网路, 也不是想上全球资讯网, 只是 想上 Amazon 网路书店.

这正是 "杀手级应用程式" 的典型定义 -- 能让使用者为了要用这个程式而去 买一台电脑.

这其中有趣的是: "杀手级应用程式" 已不再是个人桌面生产软体 (desktop productivity application), 也不是内办公室 (back-office) 企业软体系统, 而是全新的类型, 也许你可以称之为: "资讯应用程式" (information application) 或者 "资讯体" (infoware).

资讯软体将旧有运算模式不能处理的任务用新的方法解决, 几年前, 假如你想 要搜寻一个有一百万本书的资料库, 你先得把你想找的书告诉图书馆员, 他知 道既有的电脑搜寻工具的神□语法, 也许能帮你把书找到, 当时, 假如你想买 本书, 你会到书店去, 但所能选的书会比较少. 但是现在情形完全改观了, 每天都有上万人上网, 然後在有数百万笔纪录的资料库中搜寻书和买书.

这其中的奥□就是电脑朝人际沟通的方式迈进了一步, 全球资讯网式 (web-based) 的应用程式利用平易的英文组成了它们的介面: 文与图, 你不用像学习使用其他软体一样, 就能学会控制它.

传统的软体在许多程式中只内含了小量的资讯; 而资讯体却在许多资讯中内含 少量的程式. 资讯软体产品的动作非常简单: 做个选择, 买或卖, 输入少量资料, 然後获得熟悉的结果, 这些动作通常由超连结所指向的指令稿 (scripts) 完成, 而这些指令稿也许会存取一个具有成熟软体的伺服器, 但是和使用者间的交谈 仍大量藉助网页 (这些网页的作者也许是作家, 编辑, 或设计家而不是程式 设计师, 事实上网页制作所需要的程式支援很少)

资讯体的介面变化多端, 例如 Amazon 线上书店对书的展示每小时更新一次 (根据它的售价), 顾客可以立刻对书加上意见和评□, 这成为多样的使用者介面 中关键的部分. 然而在舞台之後, 有一群管理者和程式师持续地在制造这些产品. 动态展示内容不完全是自动产生的, 通常还需人工加以剪裁, 通常这部份工作会 利用许多快速而繁杂的命令稿工具 (scripting tools), 这些工具大部份都是 开放原始码.

资讯体的介面并不如其他一般软体的介面有效率, 但对於很少有机会做的事, 或每次都需要做不同处理的事, 它的长处就显出来了. 当许多资讯呈现在你面前 , 等你做决定时, 它是很好的使用介面. 不论你要在 Amazon 线上书店买书或 CD, 或是在 E*Trade 线上下单买股票, 交易的过程真的非常简单. 这正是所谓的 资讯品质 -- 和你交谈的应用程式的重心, 能帮助你做出决定.

Perl 在雅虎和亚马逊的舞台 (Perl at Yahoo! and Amazon.com)

当许多人听到 Perl 和资讯网 (Web) 时, 他们最先想到是 "共通闸道介面" (Common Gateway Interface -- CGI, 由资讯网伺服程式呼叫的外部程式, 这个程式被呼叫後会产生出网页). 但即使 Perl 是 CGI 程式最常用的语言, 但它对下一代的资讯体 (infoware) 应用程式的重要性远超过 CGI.

Master Regular Expressions 一书的作者 Jeffery Friedl 这麽说: "Yahoo! 网站上的内容并不是我们创造出来的, 而是我们收集而来", 他同时也是 Yahoo! 全职的 Perl 程式师. "我们资料的来源成千上万, 每种资料都有它们自己 的格式, 我们在资料□进 Yahoo! 前, 做了大量的处理, 理清它们或是找出它们 该放的地方." 例如, 为了让 quotes.yahoo.com 的条状活动广告能连结到适当 的地方, Friedl 需要写一个 "认名字" 的程式, 然後能搜寻 15000 间以上的公司 名称, Perl 以正规表示式 (regular expression) 分析文章的功能让这个程式 更容易写出来.

Perl 也是系统管理主流的重要工具, 用来确保网站运行正常. 大量的 Perl 程式 持续在 Yahoo! 网站及其他相关的连结网站上 "爬行", 当它们发现某个网页位置 (URL)传回的结果和过去不一样时, 会呼叫 (B.B. Call) 管理者, 这些爬□中最 有名的就是 "狞笑的死神". 假如自动连线程式连往某个网页失败很多次, 那麽 Yahoo! 就不会再列出这个网页.

Amazon.com 的网页创作环境, 显示出 Perl 结合不同工具的特异功能, 它是个 极好的 "胶合" 语言. 使用者先填表单编辑文件, 随後交由一支 Perl 的程式 处理, 它会把原来的文件转为部分完成的 SGML 格式的文件, 起始的编辑工具 可以是 Microsoft 的 Word 或 GNU 的 emacs (由使用者自行选择), 并且和 CVS (Concurrent Versioning System -- 版本控制系统) 及 Amazon.com 自家 的 SGML 工具整合在一起, Amazon.com 的 SGML 文件是为了在同一文件来源下, 网站能展现不同的风貌 -- 例如, 含图或不含图的 HTML. 还有一支 Perl 写的 语法解析器能先把 SGML 文件转成 HTML 样式, 在编辑者预览确认後才上网站.

全球资讯网正在改变整个电脑运算的典型, 当我回想 1994 年的时候, 这情形 再清楚不过了, 当时的 Microsoft 尚未投入全球资讯网, 我和该公司的副总裁 Craig Mundie 透过卫星连线, 在日本 NTT 的一个节目中同台演出, Mundie 示□他们公司的 "老虎" 视讯随选伺服器, 它的使用介面和 Windows 同出一辙, 有著一连串的选单, 而选单像是一种虚拟的遥控器.

相当明显地, 对於身陷全球资讯网的人们而言, 视讯随选的操作介面应该要 长得和 Web 资讯介面 (Web-like information interface) 一样才对, 即使 Microsoft 有一个视讯随选的绝佳介面 (如它的电影百科 CD 版 -- Cinemania), 但却无法改变这个铁的事实. 选一部电影来看最简单的作法该是怎麽样? 先搜寻 分类, 读一些影评, 看一些报导, 然後按下超连结刚始看电影, Cinemania 有上 述全部的功能, 除了最後一步以外. 超文件资讯产品的力量只有在连结到网路上 的伺服器时才释放出来, 突然之间, 资讯不是最後的目的, 而是让使用者有一个 驾驭资讯的介面, 而传统软体若要做到同样的事, 介面会复杂得多.

资讯介面 (information interface) 特别适用於 "要使用者做选择" 的应用程式 , 然而它们的意义仅在於 "只做那麽一次", 讲得更明白些, 像所谓的软体安装 精灵正是这种趋势的案例之一.

也有一些资讯应用程式采用传统软体的使用者介面, 但它们会提供动态的资讯 输出, 我最喜欢举的例子是 -- 几年前无法想像会有这样的事 -- 查地图找方向, 在 maps.yahoo.com 你可以输入两个地址 (甲, 乙), 然後就会得到一张地图, 上面标示出一组方向指引, 告诉你如何由甲地到乙地.

而这和开放原始码软体有什麽关系呢?

答案就在於这个典□正在改变电脑工业, 当某一个市场成熟时, 既有的玩家按 他们自己的路发展, 一样可以获得巨大的独占利益, 这使得他们很难接受新东西, 很难接受新的参赛者 (野蛮人, 引用 Philippe Kahn 的话) 进来开创新的市场.

当规则改变时 (As the rules change)

事实上, 既存的玩家通常是既得利益的开创者, 他们可以□试开发新市场, 但却无法完全接受新规则, 微软超越 IBM, 成为电脑工业的主宰者, 就是 一个典型的例子. IBM 的市场之所以沦陷给微软, 正是因为它没发觉到运算 的力量已经由温室 (译注: 比喻大型主机, 因以前的大型主机通常都放在 冷气房内) 移到桌面 (译注: 比喻个人电脑), 也已经由专属硬体移往一般硬体, 由硬体转向软体.

同样地, 虽然微软试图进入多种资讯商务 (information business), 但它似乎 仍然不了解 (但它以前曾了解) 软体已不再是电脑事业的主要价值, (即使微软 投注的努力以自然语言来说, 是 "资讯"(info) 多於 "软体"(soft), 好比文法 只有一些规则, 但字汇和语意表达出的资讯却有万千.)

在过去 IBM 称霸的日子, 硬体称王, 要进入电脑事业的门槛相当高, 许多的 软体都由硬体厂商, 或者硬体厂商的协力厂商提供.

随著个人电脑的普及而成为实用的平台 (也成为发展开放系统的平台, 如 UNIX), 彻底改变了这个规则, 突然之间, 门槛降低了许多, 一些如 Lotus 的 Mitchell Kapor 和微软的比尔盖兹的创业者纷纷发迹.

如果你观察全球资讯网早期的历史, 你会发现和上述类似的片断 -- 微软主宰 办公桌软体後, 已经造成进入软体事业的关卡非常地严苛, 甚至微软刻意阻挡 它的竞争者, 使得软体应用程式日益复杂, 一个在车库 (或小阁楼) 单打独斗 的程式师已不可能造成任何冲击 (译注: 苹果公司的两位创始人 -- Steve Jobs 和 Steve Wozniak -- 当初就是在车库中创造 Apple II).

对照起来, 开放原始码的模式降低了发展软体门槛, 你可以免费试用新产品 -- 甚且不仅於此, 你可以在上面建立自己想要的版本, 仍然是免费的. 在开放原始 码的模式下, 许多彼此不认识的人都可以获得原始程式码, 检视它, 如果有人不 喜欢某个特色, 他可以加入他喜欢的, 或者移除不喜欢的, 或者重写程式, 如果 他把这个修正回馈给原来程式的使用社群, 那麽这个修正就会很快地广为流传.

更重要的是 -- 因为最初的程式发展者并不著眼於商业目的, 而是为了解决真正 的问题, 而且在较宽松的环境中还有实验的空间, 就如同开放原始码社群常说的: 开放原始码软体 "解决你的切身之痛" (这和 HP 的原型工程师设计新工作台 的意义不同). 由於多方进行的发展模式, 新特色会由使用者加入, 开放原始码的 程式被设计後, 同时也开始 "进化".

事实上, 因为市场门槛和衍生产品 (能和原产品瑕□相容) 不造成负担时, 市场进化的力量会运作得更自然.

进化不造就单一赢家, 而是多元化 (Evolution breeds not a single winner, but diversity)

许多开放原始码程式明确地表现出它们的特色就是强大的力量, 也许最清楚的 例子就是 Perl 了, Perl 被喻为 "网际网路的输送带 (the duct tape of the Internet)".

Larry Wall 当初创造 Perl 只是为了把他遇到重覆性的系统管理工作自动化, 在网路上发表这个程式语言之後, 他发现有愈来愈多的应用, 同时这个语言也 成长了, 发展的方向常是始料所未及.

Perl 被喻为 "厨具百宝箱" 是因为它对其他较 "正交" (orthogonal, 译注: 指做一件事只有一种方法) 的电脑语言的设计者而言是几乎 "含糊" (mess) 的. Larry Wall 在最近 "第二次 Perl 研讨会" (Perl Conference 2.0) 中 提到了这一点, 他展示了一张看起来像是小朋友涂鸭的图画, 然後解释道:

"这张图代表许多事. 它是一张空气分子运动图, 它是一张经济学的图, 它是一张 显示出人类语言看起来像什麽的图, 它是一张你公司资讯系统的图, 它是一张 全球资讯网的图, 它是一张浑沌的图, 也是一张复杂度的图..."

"这很重要, 但却不容易理解, 英文之所以有用, 是因为它 "含糊", 正因为它 "含糊", 所以能够把问题 (也是含糊) 阐释好, 同样地, Perl 也被设计成 "含糊", 不过是用最好的方法来达成.

就如何让发展者们致力於程式特色的开发而言, 开放原始码发展模式是一种非常 有效率的方式, 新软体的发展和客户的需求构成了一个紧致的反馈回圈, 市场的 冲击或者上司的采购决策并不会扭曲新软体, 发展自基层的软体最适合用来解决 基层的问题.

因为利用开放原始码的软体当作全球资讯网网站的重心, 和它比较简单的发展 模式, 企业家们如杨致远和 David Filo 才可能开创出他们的事业. 世界上最大 和最成功的网站 Yahoo! 不是偶然造成的, 而是建□在 FreeBSD 作业系统, Apache 和 Perl 这些可自由取得的开放原始码软体之上.

改变这个游戏 (的规则) (Change the [rules of the] game)

就如同过去一般, 电脑工业的新阶段就是上一阶段的实用化, 像 RedHat (Linux 厂商的佼佼者) 的 Bob Young 曾指出他的目标并不是打倒在作业系统市场占有率 第一的微软, 而是缩减作业系统市场的总值 (译注: 即让作业系统的价格更低廉).

开放原始码在这场游戏中并非要打击微软, 而是要改变这场游戏的本质.

的确, 对市场付出天文数字般的投资的资讯提供业者, 如 Amazon 和 Yahoo! 仍 常被人们拿来和微软比较, 但事实写在萤幕上 - 人机交谈介面, 将过去未曾电 脑化的工作电脑化的机会, 出现在资讯体 (infoware) 中而不是软体 (software) 中.

当新的 "杀手级应用程式" 出现时, 软体的角色将逐渐加重成资讯体的引导者. 我们在这里必须注意当电脑工业的中心由硬体移到软体後, 其实硬体并未式微, IBM 仍然是一间生意兴隆的公司 (虽然它许多竞争者的规模变大或缩小), 但 是其他硬体厂商的出现, 如: Dell, Compaq, 特别是 Intel, 显示出它们更适应 新的游戏规则.

Intel 体认到它们真正的机会不在於赢得电脑系统的战争, 而在於成为这场电脑 战争中的军火商.

开放原始码软体的大挑战并非在办公桌上取代微软而成为霸主, 而是它是否能 精研出一个商业模式, 使得新一代的电脑应用程式挂上像是 "Intel Inside" 这样的商标 (译注: 即 "Open Source Inside").

在其他方面, 就如同 Digital Research 之於个人电脑作业系统 (译注: 这里 指的可能是之前 Digital 的研发团队到微软发展 Windows NT), 了解这个商机 的厂商将延揽开放原始码的先锋.

同时这也指出 -- 开放原始码软体已经在电脑工业发展的路上生出分枝, 就如同 微电脑早期的拓荒者奠定今天电脑工业的格局, 开放原始码软体开创新格局的 剧本正要展开, 它将在下一个五到十年, 根本地引导电脑工业转型.

新文艺复兴的科学 (The Science of the New Renaissance)

Linux 的创始人 Linus Torvalds 曾提及他名字 "Linus" 的由来, 是因为他父母亲敬佩获得过两次诺贝尔奖的 Linus Pauling, Pauling 的第二个诺贝尔奖是因为他的研究导致 Crick 和 Watson 发现了 DNA 的结构.

Linus 这个名字取得好, 科学毕竟就是开放原始码的事业, 科学方法基於发现 及判断的过程, 若要验证科学的结果, 则该结果必须可再现, 让结果再现的 唯一方法就是将研究过程公开出来 -- 包含假设, 实验条件和实验结果. 同一个发现可藉由许多不同路径来触发, 常常许多科学上的发现是不同人在独立 工作下完成. 但是最後必须把发现过□中的资讯分享出来: 使得其他尚未得到 结果的科学家也能完成, 并且播下尚未出现的新观念的种子, 让这些研究可以 日新又新.

科学家讨论 "结果再现" (replication), 开放原始码程式师讨论 "除错" (debug), 科学家谈论 "发现" (discovering), 开放原始码程式师谈论 "创造" (creating). 到最後, 开放原始码运动其实只是科学方法的延伸, 因为电脑工业 的重心就在於电脑科学, 就 Crace Hopper (编译器的发明人) 的话来说: "对我而言, 写程式不只是一项重要的实用艺术, 它也是基础知识的大事业."

虽然在当时 Pauling, Crick 和 Watson 并未发觉他们正站在生物科学新时代的 入口, DNA 的双螺旋架构被发现时, 生物学和化学可说是一个行业, 一门实际的 艺术, 这项发现工作是由数个少数人组成的团体, 在学术研究经费的赞助下完成 的, 然而改变日後和工业间关系的因子已被种下, 许多医学上的突破, 如小儿 麻痹疫苗和盘尼西林的发现, 使得生物的科学转变为工业.

今天的有机化学, 分子生物学, 和基础医药研究已不是少数从业人员的事业, 而是工业的附属, 虽然在学术界仍进行著以上这些学科的研究, 但绝大多数的 研究者和绝大多数的研究经费都来自於制药工业, 这种情况下, 科学和工业间的 合作充其量只是一种不稳定的合作关系, 也许制药公司能够赞助学术单位进行某种 程度的 "梦想" 研究, 但它们要能从中获取一定的利益, 想想看, 制药公司会 赞助无药疗法研究还是用药疗法研究?

开放原始码如同科学方法 (Open source as scientific method)

电脑科学也和工业间存在著不稳定的合作关系, 早先由学术界电脑科学家带来的 新技术, 现在由电脑工业则把这些新技术再向前推, 虽然目前参与开放始码运动 的程式师中, 许多都是散布世界各地的电脑科学学系的大学生或研究生, 但工业 界却有越来越多的程式师参与开放原始码运动, 越来越多的公司以开放原始码的 模式为基础, 来发展业务.

基本上计算机科学和其他科学不同的地方, 在於它只有一种方法让其他人也能 得到相同的结果, 就是 -- 公开分享原始程式码. 为了向某些人证明你的程式 是正确的, 你必须让他们能编译并执行你的程式, 所以开放原始码的发展模式 就是科学方法的延伸.

"可再现" 的特性使得科学实验的结果更可信, 光一位科学家没办法估计所有可能 的测试状况, 但也不需要拥有一个可以做到完全测试的实验环境, 藉由对研究社群 公开假设和实验结果, 成群的目光会照到一双眼睛可能错过的地方. 在开放原始 码的发展模式中一样有类似的法则 "足够多的人来看程式, 错误消失无踪" (Given enough eyes, all bugs are shallow). 藉由公开原始程式码, 开放 原始码的发展者让软体更加稳固, 程式在更多, 更广的条件下被测试, 某些人没 发现的错误却被其他人发现了.

科学实验结果的公开促成了更多的新发现, 科学的方法将重覆努力的机会减到 最低, 因为科学家会知道是否有人从事类似的研究, 然而进步却不会终止, 因为 当一位科学家停止一项研究时, 如果这个研究的结果是值得的, 那麽就会有其他的 科学家继续研究下去. 同理, 在开放原始码的发展模式中, 原始码的公开使得 软体创新更为容易, 继承未完软体计画的程式师, 便可以藉其他的计画的原始程式 来提携未完的计画, 或者和其他计画合并以求整合资源. 有时候一个开放原始码 软体的计画会激励出另一个计画, 一个有价值的计画在负责的程式师离开後, 并不会成为孤儿, 因为原始程式码都还在, 其他程式师可以一步步跟上来. 例如, 开放原始码社群自豪的几个程式: BIND, Fetchmail, 和 GIMP (GNU 的影像处理 软体) , 都曾经发生原先的计画负责人中途退出, 但计画却仍後继有人.

今天的开放原始码发展模式其实早在十几年前就已根植於计算机科学的学术界, 但为什麽今天开放原始码能如此蓬勃发展呢? 这是靠网际网路能迅速传播资讯 的功能, 当 Watson 和 Crick 发现双螺旋结构时, 他们可以预期这项资讯由 英国剑桥传到美国加州理工学院, 只要几天或最多几个星期的时间, 在今天 资讯的传播几乎是即时的, 拜网际网路之赐, 开放原始码注定要在数位革命中 诞生, 相似地, 因文艺复兴期间发明了印刷术, 产生了现代科学.

网际网路标准制定委员会也开放原始码 (The IETF as Open Source)

开放原始码运动中最好的模□之一就是技术世纪的惊奇 -- 网际网路标准制定委员会 (Internet Engineering Task Force 简写成 IETF), IETF 是 Eric Raymond (教堂与市集一文的作者) 市集主义的最佳证明.

IETF 所定出的标准规格促成网际网路的诞生, 制定标准的过程多是藉由通信 论□ (mailing list) 的讨论, 各工作群组面对面的讨论一年只举行三次, 任何人都可以参加 IETF 的通信论□及讨论会, 任何人都有发言权 (但如果你 没有准备, 或是没有团体支持, 那麽你的声音会被忽略或被其他声音压过). 长期参与 IETF 的哈佛大学人士 Scott Bradner 指出: "IETF 中没有投票, 只有试著去了解意见一致的程度, 可以透过举手, 低哼示意, 或其他主席认可的 方式 -- 但是决议必需获得大多数人的同意, 51 比 49 不算数 -- 必需是 80 比 20 或 90 比 10, 但也不需要全数同意."

在 IETF 中, 分组总监 (Area Directors) 和小组主席 (Working Group chairs) 具有阶层式的组织, 然而, 大致说来, IETF 的运作模式就如 Dave Clark 所描述 的 "没有王者, 没有尊者, 只有大略的意见一致和可执行的程式."

上面说到的最後一点是很重要的, IETF 是一个由下而上的标准制定团体, 有时候, 分组总监定出问题, 之後徵求志愿解题者, 更常见的情形是: 一群对某个 题目有兴趣的人员组成一个小组, 然後争取其他人的认同. IETF 的成员靠他们 技术上的长处, 去评估这些互别苗头的解决方案, 评估的重点在於是否简单 (simplicity) 与是否谐和 (interoperability).

当然, IETF 内部存在有许多政治性的问题, 这需要靠各个小组的主席竭尽所能, 以维持标准制定过程有进展. 除此之外, 就像其他许多成功的开放原始码专案 一样, IETF 有一个基本的远见在引导他们, 这些远见大部份都在网际网路结构 布告 (Internet Architecture Board) 上有详细的描述. 但是这些小组主席和 结构远见者并非由任何中央机关指派, 他们是在寻找最佳解决方案的过程中产生 的 (随後受同侪推派).

难怪微软在它不名誉的万圣节文件 (Halloween Document) (万圣节文件是一份 微软内部的备忘录, 不知为什麽流出至 Eric Raymond 手上, Eric Raymond 把它加以分析後, 反应至开放原始码运动中, 该文件中将开放原始码软体简称 为 OSS) 认为 IETF 和 Linux 一样, 都是它们要化解的威胁:

"在 IETF 不同的任务小组中, 都拥有大量脑力的投入, 这使得他们能快速地建出 结构模型 (architectural model), 并整合入 OSS 的专案计画..."

"OSS 许多专案计画在伺服器的应用上都能占有一席之地, 这是因为他们有众多而 实用的工具程式, 和简洁的协定, 藉由扩充这些协定和发展新协定, 我们可以挡 住 OSS 专案计画进入市场的入口."

由於中古世纪缺乏健全的资讯传播骨干, 手写的文件需靠手抄如此高昂的 代价才能复制, 所以文本身必需具有及时的价值, 如贸易纪录, 银行交易, 外交书信都简洁到能传送出它及时的价值. 至於炼金术士, 僧侣, 和哲学家 (这些人後来被称做科学家) 的手稿则只有非常低的优先权, 所以其中的资讯传播得很慢. 印刷术改变了 这一切, 将进入资讯传播骨干的门槛降到很低, 之前独力研究的学者们, 第一次 可以和全欧洲的其他学者成立一个研究的社群, 当然这个社群的建立有赖於 公开分享资讯.

这样的社群的诞生可说是因为学术自由的观念, 後来我们把这个社群的研究过程 称作科学方法. 如果没有形成社群的需求, 就不会有这些事发生, 藉由分享资讯 使得科学社群一同研究, 这种情形已经有好几个世纪了.

想像一下, 假使牛顿保留了运动定律, 而且在三十年的战争内, 藉由和炮兵签订 国防合约, 把运动定律商业化. "我不会告诉你我如何知道抛物线运动轨迹, 但 我要量一下你炮管的口径, 然後收费" 当然这个想法听起来挺荒谬的, 虽然科学 并没有这麽演进, 但它的确不能这样演进, 独门秘方会使得科学的发展窒碍难行.

网际网路就是数位时代的印刷术, 又一次大幅降低了进入资讯传播骨干的门槛, 原始程式码不再需要用纸带散布 (如在最原始的 UNIX 中执行的程式), 不再 需要用软碟片散布 (如在早期的 DOS 时代), 甚至也不需要用光碟散布. 任何 FTP 或 Web 的伺服器都可以成为便宜和立即的散布站台.

虽然开放原始码运动带来许多希望, 但我们不能忘记好几个世纪以来科学的遗产 是开放原始码发展模式的根基, 今天电脑科学和电脑工业彼此间的关系并不稳定, 为了短期内的经济收益, 工业巨人迫使公司发展新的专利产品. 但是, 当越来 越多的电脑科学研究工作由工业界发起而不是学术界时, 工业界必需藉公开分享 创意, 也就是开放原始码发展模式来培育学术界. 电脑工业厂商这麽做并不是 为了要利他或是其他更伟大的动机, 而是为了最现实的理由: 为了自己的商业 利益.

首先, 对开放原始码界最好的程式师而言, 金钱上的回馈几乎不是他们主要的 关心事项, 这些人参与的是一场信誉的竞赛, 历史上也显示出科学上的成功 要比经济上的成功更为持久, 我们记得过去一百年来一些有名的企业家, 如: 卡耐基, 洛克斐勒, 和福特, 我们记过去一百年来更多的科学家或发明家, 如: 爱因斯坦, 爱迪生, ..., 和鲍林, 当这个年代 (由现在起一百年) 的历史被 写下, 後人在电脑工业中也许除了记得比尔盖兹, 其他人就记不了几个了, 但他们却很有可能记得: Dave Clark, Bill Joy, ..., Larry Wall 或 Linus Torvalds.

其次, 更重要的是, 工业需要来自科学的创新. 开放原始码能以科学的速度和 创造力来发展软体并对它除错, 电脑工业需要新一代的点子, 这些点子将来自於 开放原始码的发展.

为了支持 "数位文艺复兴" (digital renaissance), 我们需要开放原始码, 开放原始码不仅为电脑科学带进步, 也帮电脑工业向前迈进.
