1. 教堂与市集 (The Cathedral and the Bazaar)

Linux 打破了许多软体发展的传统, 这个世界级的作业系统在五年前仅仅靠著 如丝般的网际网路, 神奇地联合了散布在全世界数以千计兼职的玩家们来发展 它, 谁曾料到会发生这样的事情呢?

我当然也没料到. Linux 出现在我电脑萤幕是在 1993 年年初, 当时我埋首於 UNIX 及开放性原始码的软体发展已有十年, 1980 年代中期, 我是 GNU 专案 首批的贡献者之一, 我写过许多开放性原始码的软体放到网路上供人使用, 也 曾独立或协同发展好几个程式 (nethack, Emacs 的 VC 和 GUD 功能, xlife, ... 等等), 这些程式到今天仍广泛地为人所用, 我想我知道这是怎麽办到的.

[译注] GNU 是自由软体基金会 (Free Software Fundation) 的一个专案, 目标是发展出 UNIX 上所有程式的自由版本. Emacs 是自由软体基会发展出来的一支程式, 可做文字编辑器, 提 供写程式的整合发展环境, 用来读电子邮件, 新闻群组, 甚至浏览 网页. 更详细的资讯请参考 http://www.gnu.org.

Linux 扭转了许多我认为我已知道的观念. 多年来我一直被传以: 小工具集, 快速原型发展及程式进化的 UNIX 福音. 但我也相信对於有一定复杂度的程式 必需使用集中和有经验的方法来开发, 我相信最重要的软体 (作业系统以及庞 大的工具程式如 Emacs) 必须如建造一座教堂般, 由个别的高手或一小群专家 在光辉的孤立中小心翼翼地精雕细琢, 时机未到之前, 不会释出测试版.

Linus Torvald 的软体发展风格 (尽早并经常发表新版本, 授权每一件作者可以 委托的事, 不拒绝几乎到混乱程度的程式) 的出现如同一个惊奇, 没有令人肃, 然起敬的教堂, 甚至 Linux 的同好们似乎组成了一个有不同流程和不同方式的 大市集 (Linux 的档案服务站台就是它适切的象徵, 每个人都服从著自由 的规则), 以这个风格发展出来的 Linux 既一致又稳定, 表面上看来 真是一连串的奇迹.

[译注] Linus Torvald 是 Linux 核心程式 (kernel) 的原始作者.

市集模式似乎是可行的, 并且运作得很好, 这个事实带来了相当的震憾. 当以 我的方法去认知时, 我除了努力做好个人的专案, 并也试著去了解为什麽在 Linux 的世界, 不但没有因为浑沌不清而四分五裂, 反而以教堂建造者几乎想像不到 的速度在茁壮.

直到 1996 年年中, 我想我才开始了解这一件事. 我得到了一个绝佳的机会 来试验我的理论, 这个机会是一个开放性原始码型式的专案, 正好可以试用 市集模式来发展, 所以我做了这个专案, 而且更有意义的是它成功了.

接下来在这篇文章中我将陈述这个专案的故事, 并且以它为例提出关於有效 地利用开放性原始码模式来发展软体的格言, 这些法则并非都是我在 Linux 中第一次学到, 但我们可以看到 Linux 的世界是怎麽赋予它们特别的意义. 如果我是对的, 那麽这些格言将会帮你真确地了解是什麽促成 Linux 社群 成为好软体的原创者, 并且帮助你变得更具生产力.
