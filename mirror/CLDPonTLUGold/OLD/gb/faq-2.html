2. SGML 相关问题

2.1 什麽是 SGML? 为什麽 Linux HOWTOs 要用 SGML?

答: SGML 是 Standard Generalized Markup Language 的简称，它允许你指定文件的结构 --- 也就是，用何种事物来标示文件。 SGML 使用"文件形态定义"(Document Type Definition, DTD)来指定文件的结构。而 Linux HOWTOs 采用一种叫做 linuxdoc 的 DTD 做为文件的结构。 详细的说明，请参考 SGMLtools User's Guide 中译版。

采用 SGML 做为 Linux HOWTOs 的写作格式，是因为它可以转换成为多种的输出格式，包括了纯文字，HTML，LaTeX, PostScript 等格式。最重要的，它为所有的 HOWTOs 文件提供了一致的外观，便於管理与维护。请参考 这里的说明。

CLDP 建议所有中译的 SGML 文件开头加上译者及翻译日期，如: 

<author>
作者: John <tt><htmlurl url="mailto:john@linux.org"
           name="john@linux.org"></tt><newline>
译者: 张三 <tt><htmlurl url="mailto:chang3@linux.org.tw"
           name="chang3@linux.org.tw"></tt>

<date>v1.0, 1 January 1998
翻译日期: 28 July 1998

2.2 我不了解 SGML, 我可以翻译 Linux HOWTOs 吗?

答: 是的! 当然可以。您不需要是 SGML 的专家才能做这件事。事实上，由於您也不是从头开始写一份文件，您甚至也不需知道共有那些 SGML 的标签(tag). 您所要做的，就是先看一遍 SGMLtools User's Guide, 认识什麽是 SGML 的标签。 然後拿起您要翻译的 SGML 档案，翻译其中的英文句子，而保留所有的 SGML 标签即可。

简单的说，SGML 标签就是由 < 及 > 所包含的英文字组成的。 另外一种特殊符号，是由 & 开头而以 ; 结尾的英文单字。 因此，千万不要将这个标签 <author> 翻译为 <作者>, 或将 <date> 翻译为 <日期>. 还有包含在 <!-- 及 --> 之间的文字代表注解，可以不必翻译。

另一需要注意的地方是交互参考的标签 label 及 ref. 其中的 'id=' 内的字元只是参考用的符号，不需翻译。 而 'name=' 内的文字为说明用，可以翻译。例如: 

<ref id="reference6" name="Reference 6">
可译成 
<ref id="reference6" name="参考资料六">

2.3 我翻译的文件一定要采用 SGML 格式吗?

答: 如果您要翻译的是 Linux HOWTO, 那麽答案是 "对的!" 您一定要采用 SGML 格式。 因为所有的 Linux HOWTOs 都已采用 SGML 格式，原因也 如前所述。 而且所有 CLDP 的文件转换工作都已自动化。如果您寄给我非 SGML 格式的文件，我无法处理，只好退回给您。

如果您要翻译的是 mini-HOWTO, 那麽也可采用 HTML 或纯文字格式(并非所有的 mini-HOWTOs 都已采用 SGML 格式)。 但若您要翻译的 mini-HOWTO 也是采用 SGML 格式写作，建议您还是也用 SGML 较适当。

请不要害怕。用 SGML 格式并没有想像中的困难。请参考前面的解说。

2.4 如何转换我的中文 SGML 文件?

答: SGMLtools 尚不能处理远东地区的双字元语系。因此除了安装最新版本的 SGMLtools, 您还需要安装 SGMLtools 中文套件 zh-sgmltools. 请在下面地点取得:

ftp://ftp.linux.org.tw/pub/CLDP/src/
zh-sgmltools-1.1.tar.gz
zh-sgmltools-1.1-x.i386.rpm
zh-sgmltools-1.1-x.src.rpm

安装很简单。解开後 

make
make install

详细的使用方法，请参考 SGMLtools 中文套件使用说明。

另外如果您手边也有 FreeBSD 2.2.x 或 3.0 的机器，有一个 FreeBSD package 'sgmlformat' 也可处理中文的 SGML 档案。请依序安装下面的 packages:

  • perl-5.00404
  • iso8879-1986
  • linuxdoc-1.0
  • docbook-3.0
  • jade-1.0.1
  • sgmlformat-1.6
(以上的 packages 皆可在 freebsd.csie.nctu.edu.tw 找到。)

然後将 /usr/bin 目录下的 sgmlfmt, sgmls, instant 改名或删除，并将 /usr/share/sgml 整个目录改名或删除，这样就可以用新的 sgmlformat 了．

再来修改您的文件，将 DTD 的定义 

<!doctype linuxdoc system>
改为 
<!DOCTYPE linuxdoc PUBLIC "-//FreeBSD//DTD linuxdoc//EN" []>

要转换为 HTML 格式，用 

sgmlfmt -e big5 -f html Foo-HOWTO.sgml

要转换为纯文字格式，用 

sgmlfmt -e big5 -f latin1 Foo-HOWTO.sgml

但目前 sgmlformat 仍无法转换 LaTeX, dvi 及 PostScript 格式。
