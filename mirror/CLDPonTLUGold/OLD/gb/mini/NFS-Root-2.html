2. 一般概论

一般来说, 对这样的工作站会有下列的问题:

  • 系统必须知道他专用的 IP 位置, 并且视需要而重设 Ethernet 的组态.
  • 系统必须知道 NFS 伺服器的位置, 与根目录的挂载路径.

目前核心 (1.3.7x) 对於 NFSROOT 的实作提供了下面的解决方案:

  • IP 位置可以利用 RARP 取得; 完整的 Ethernet 组态则可以用 LILO 或 LOADLIN 的参数传递给核心.
  • 要被挂载的 NFS 路径可以利用核心的参数来传递. 如果没有在此指定, 核心会假设 RAPR 伺服器同时也是 NFS 伺服器, 并且用核心编译时的预设路径. (目前的预设路径是 /tftpboot/< 机器的 IP 位置>.
  • 客户端的组态也可以利用 BOOTP 取得.

在开始设定无磁碟环境之前, 你必须决定是否要使用 LILO 或 LOADLIN 来启动系统. 用这种方式的好处是较有弹性, 缺点则是速度较慢. 不使用 LILO 来开机的速度会较快. 你必须自行决定.
