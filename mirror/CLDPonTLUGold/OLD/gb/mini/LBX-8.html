8. 如何使 LBX 开始工作？

OK，接下来的步骤都很简单。 将下列 LOCAL 以及 REMOTE 用你本端以及远端的主机名称取代。 ( 千万不要搞混了 )On LOCAL:

  1. 启动你的 X server。
  2. 告诉你的 X server 哪些远端系统是被允许存取的。 使用 host-list 的方法，键入 xhost + REMOTE。如果你要使用 xauth 你需要作更多事情；请参阅 man xauth(1) 以获得更多的资讯。 如果你不熟悉 X 远端的存取，那麽你需要参阅 Remote X Apps Mini-HOWTO 。

On REMOTE:

  1. 起始 lbxproxy 并告诉它传送讯息到本端的 X server，如下∶ 
      $ lbxproxy -display LOCAL:0 :1 &
    这告诉 lbxproxy 在远端系统使用 display :1 ；如果系统大於 1 的 display 已经在使用。你可以使用 :2 之类的设定取代。
  2. 设定你的 DISPLAY 环境变数，使此环境变数和你 lbxproxy 所设定的相同。取代平常的 DISPLAY 设定 
      $ DISPLAY=:1
  $ export DISPLAY
    如果你使用 csh 之类的，以下可能是你所需要的∶ 
      % setenv DISPLAY :1
  3. 如果你使用 xauth 你需要确定你的 cookie 正确的运作。你可以从 Remote X Apps Mini-HOWTO 获的更多的资讯。
  4. 执行你的 X 程式！

接下来就完成了；所有在 Display 环境变数为:1 的 X 程式都会在都会使用 LBX。当然，你没有办法起始一个 X 程式，其 Display 的环境变数是指向 LOCAL:0 以及在同一个时间下执行。
