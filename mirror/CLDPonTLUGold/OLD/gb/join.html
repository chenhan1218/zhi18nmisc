如何翻译 Linux HOWTO 或 mini HOWTO

  1. 首先，当然您要会读写中文，也要看得懂点英文啦! 不过也别紧张! 您不需要托福考六百分以上才能翻译 Linux HOWTO。事实上大部份的英文都算简单的。 因为，许多 HOWTO 作者的母语也不是英文呀!

  2. 阅读一下 CLDP 常见问题与解答，很多您想知道的事情都已经写在里面了。

  3. 再来，选一篇您看起来有兴趣，或有些研究的 HOWTO 或 mini HOWTO 文章为目标。 至於到底有那些 HOWTO 或 mini HOWTO 文章，请看 HOWTO INDEX。

  4. 确定您选择的目标还没有被人翻译过。请参考翻译中文件列表以及未翻译文件列表。 您最好先与我联络一下以确定您要翻译的文件还未被人"认领"。

  5. 所有的 HOWTO 文件版权都是属於原作者的。为了礼貌，最好先取得原作者的翻译同意权。这样将来文件有更新时或许也会通知您。 我想绝大部份的作者应该都会欣然同意您翻译他的文章。

  6. 如果您想翻译的是 HOWTO 文件，必须写成 SGML 文件，如此才能转换为各种输出格式。 所以您必须先了解什麽是 SGML。请先参考 SGMLtools User's Guide。 您不真正需要了解到底什麽是 SGML (其实我也不是很了解…:p)，但您应该要能看得懂 SGML 文件中的 SGML 标签，如此在翻译时才能分辨何者是文章内容，何者是 SGML 标签。 至於 mini HOWTO 则有纯文字以及 SGML 两种格式。如果是用 SGML 的，当然建议您也使用 SGML。下面所说的，都是假设您要翻的是 SGML 文件。

  7. 取回您想翻译文件的 SGML 原始文件。最简单的办法就是写信给我，我会寄给您。

  8. 如果您要自行取得也可以。但是请务必要取得最新版本的文件，大部份人都不会想看已经过时的文章。 您可在下列地点取得:

    • Metalab 原始站
    • 国内映射站: 中山 FTP | 中正 FTP

  9. 如果您不确定取回的文件是否为最新版，可与 HOWTO INDEX 上的更新日期比较。 某些文件还会注明最新版本的发表站台，您也可从该处取回。

  10. 再来就是用力的翻译啦! 不过还是有几点请您注意:

    • 请尽量保持原始 SGML 文件的格式，包括章节，顺序，甚至一些特殊标签(如 <tt>, <em>)，URL 等等。 这点其实很容易办到。您只要将原始 SGML 文件中的英文翻成中文，其它的 SGML 标签都别去动它就好了。
      例如原文为:
      For dialin and dialout use, you <EM/have/ to set up your modem a certain way (again, using <TT/AT/ commands on your modem):
      翻译为:
      要用来接受拨接以及拨号出去的话，那麽你<EM/必须/确实地设定你的数据机(再一次对你的数据机使用 <TT/AT/ 指令群):

    • 您不需将翻译的 SGML 文件排版整齐，因为这没有意义。将 SGML 转为其它格式时会自动为您排版。 不过仍有一点要小心，就是最好不要在中文句子中间换行，否则转为 HTML 格式时它会在中间加上空格。 例如若您在 SGML 中写: 

      这是一个很
长的句子。

      转成 HTML 後可能会变成

      这是一个很 长的句子。

      这样变得不好看。因此换行的地方最好在句末。

    • 在中英文夹杂的句子中，英文字与中文间前後最好加上空格比较好看。例如:

      Linux,FreeBSD与Solaris都是PC上的UNIX作业系统。

      看起来就没有

      Linux, FreeBSD 与 Solaris 都是 PC 上的 UNIX 作业系统。

      这样好看。当然，这纯属个人意见，您无须遵守。

    • 人名，地名，公司名等专有名词以不翻译为原则，直接以原文写出较好。

    • 其它的相关问题也请见 CLDP 常见问题解答 的说明。

  11. 翻译完成後请先测试您的 SGML 原始文件文法是否正确，可否顺利编译。您可安装 SGMLtools 中文套件 zh-sgmltools. 请参考 CLDP 常见问题解答 的说明。

  12. 测试无误後请将 SGML 文件寄给我。我会将它转换为各种格式并放在适当位置。 当然，不会忘了在翻译列表上加上您的大名。

  13. 之後，每当您翻译的文件有了新版本，我会主动将更新版本寄给您。然而，请保留您翻译过的英文 SGML 原始文件。 当您取得更新版的文件时，只要用 diff 指令就可很快找出更新的部份。
    diff -uN Foo-HOWTO.sgml.old Foo-HOWTO.sgml
    当您更新文件後记得再将它寄给我。

  14. 再来呢…? 再来就没啦! 简单吧? ^_^
