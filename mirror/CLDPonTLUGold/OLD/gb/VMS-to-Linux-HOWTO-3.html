3. 档案编辑

Linux 没有 EDT 编辑器，但是有很多的编辑器选择可供使用。 在所有UNIX 系统中，唯一保证一定有vi 这个编辑器---算了吧， 您的系统管理员一定已经安装比 vi 更好用的编辑器了。可能比较受欢迎的编辑器是 emacs，这可以用来模拟 VMS 上的 EDT ; jed 是另一种类似 EDT 的编辑器。

这两种编辑器用来编写程式码相当的实用，因为她们有两个 EDT 没有的功能： 高亮度相关字（syntax hilighting）和自动缩排（automatic indentation）。 而且，您可以不需要跳出编辑器，就可以直接在编辑器里编译（Compile）您的程式。 (command ESC-X compile); 如果有语法错误，游标将会自动跳到有错误的地方。 我敢打赌您将不会想再看到那只有蓝色的 EDT 了。

如果您有 emacs: 启动她，然後输入 ESC-X edt-emulation-on。 键入 ALT--X or ESC-X 可以跳出 emacs，像 EDT 的 CTRL--Z。 从现在开始，emacs 就像 EDT 一样，除了一些指令不太一样。 不同处：

  • 不行 使用 CTRL--Z 来执行命令列指令。 (如果您做了，您的 emacs 将会停止工作， 不过您可以试著输入 fg 来继续您的 emacs);
  • Emacs 有大量的线上说明（on-line help）。按 CTRL-H ?, 或是 CTRL-H T 启动。;
  • 存档： CTRL-X CTRL-S;
  • 离开： CTRL-X CTRL-C;
  • 新增一个档案： CTRL-X CTRL-F, 然後 CTRL-X B 可以在个档案中作切换。

如果您有 jed： 可以请您的系统管理员恰当地设定 jed 。 当您启动她後，模拟器就已经自动载入。可以使用 CTRL--H CTRL--H or CTRL-? 启动线上说明。指令部份，大致上和 emacs 相同。 此外， 还有一些易於操作的操作键没有被装订在原来的 EDT 中; 这些操作键可以依您的个人喜好自行设定，请询问您的系统管理员。

或许您可能会使用另一个完全不一样操作界面的编辑器。但是 emacs 直觉上会是一个不错的选择。 另一个十分受欢迎的编辑器是：joe, 她可以模拟其他编辑器，像 emacs 。 (甚至更容易上手) 或是 Dos 的编辑器。 启动这个编辑器就像 jmacs 或是 jstar 一样， 可以使用 CTRL-X H 或是 CTRL-J 得到丰富的线上说明。 emacs 和 jed 比 EDT 更具有威力。
