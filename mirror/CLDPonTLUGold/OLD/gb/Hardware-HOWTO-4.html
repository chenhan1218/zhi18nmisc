4. CPU/FPU

Intel/AMD/Cyrix 386SX/DX/SL/DXL/SLC, 486SX/DX/SL/SX2/DX2/DX4 都被支援。 Intel Pentium, Pentium Pro 和 Pentium II (基本上它是有 MMX支援的Pentium Pro) 也可以使用。 AMD K5 及 K6 可正常工作，虽然要避免使用有小臭虫的旧型 K6 。在 BIOS 设定□的 "internal cache" 这项设定关闭 就可以解决这个问题。

还有 IDT Winchip C6-PSME2006A 微处理器也是被 Linux 支援的。

Linux 有内建的 FPU 模拟器，如果你没有数学辅助处理器的话。

试验中的 SMP (多 CPU) 已在 kernel 1.3.31 (含)以上所支援。 详情及最新情况请察看 Linux/SMP 计画 网站。

  • Linux/SMP Project
    http://www.linux.org.uk/SMP/title.html

有一些早期的 AMD 486DX 会在某些情况下当机。但新的晶片应该没这个问题，况且取得一个新的晶片来换 掉旧的 CPU 应该不是问题吧!

ULSI Math*Co 系统在 FSAVE 及 FRSTOR 指令上有点问题，会导致作业系统在保护模式时发生问题。有些 旧型的 IIT 及 Cyrix 晶片也有这个问题。

在很旧的 kernels. (1.1.x) 用在UMC U5S 晶片上时会造成 TLB flushing 。
