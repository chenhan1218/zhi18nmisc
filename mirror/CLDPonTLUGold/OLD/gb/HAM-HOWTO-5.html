5.15 CLX Packet DX Cluster Program

作者

Franta Bendl, DJ0ZY, 和 Bernhard ("Ben") Buettner, DL6RAI, root@dl6rai.muc.de

简单说明

运行在 Linux 上的 PacketCluster。

当前开发状态

版本：2.07

更新日期

1997年4月1日

系统需求

Linux 2.0 或以上版本，TCP/IP 网络, SysV IPC，和一些选件。

详细说明

PacketCluster 在 7 年前就可以使用了，最早的程序是由 Dick Newell（AK1A）写成，是一个运行在 DOS 上的版本。CLX 是一个广泛的 PacketCluster 节点管理程序，能够辨识外部用户的命令，可以同时连接普通 PacketCluster 节点和 CLX 节点，所以它在组建 PacketCluster 网络时很有用。

CLX 不是一个用户程序，它是网络节点软件。对个人的无线电爱好者没有太大的作用。对 PacketCluster 和 Linux 有经验的 PacketCluster Sysop 会发现本软件的有趣之处。

下面是 CLX 的一些特点：

  • 从用户的观点看它与 internal PCxx 协议完全兼容，并允许多用户和多优先权。
  • 本软件能检测 DX 环路以避免产生无限循环的信息。CLX 过滤器能过滤掉这些重复的信息。并允许我们从多个点连接到网络，这在以前被称为 “Cluster Loop”。CLX 能分辨出所谓的“主动连接”和“被动连接”。这就使 CLX 的用户能获得更多的 DX 信息。
  • 是你的应用程序的一个 CLX 的扩展接口。
  • 通过连接脚本扩展连接兼容性。现在 CLX 能使用 TCP/IP,UUCP 或其它通信程序建立、维护实际的连接。
  • CLX 使用现代编程概念来构造程序，象为公用数据共享内存，用 LEX 和 YACC 来执行命令语言，公用程序库使二进制代码变得很小，为进程间的通信建立远程过程调用。严格的模块化使每一个模块都有有用的进程为主要任务服务。
  • 构成的模块作为分离的几个进程来运行。

5.16 DPTNT Terminal and DPBOX BBS package. -- 终端及 BBS 软件包

更新日期

1997年4月1日

系统需求

支持 WA8DED 主机模式或 KISS 协议的 TNC。

详细说明

TNT 是一个全功能的主机模式的终端程序，在 Linux 下编译它仅仅是解开压缩文件 然后敲入 `make'，它提供了哑终端和 X11 两个版本，以及很多很有用处的文档， 主要特性如下：

多任务：

TNT 在多个虚拟屏幕上支持多个 Packet Radio 的连接，你可以随意切换所需要的屏幕。每个任务都提供一个分割窗口（被分成发射窗口和接收窗口）并有一个状态条。命令可以在命令操作方式输入，也可以在终端操作方式通过命令键输入。每个虚拟窗口都可以比实际的物理窗口大并且可以卷屏。

数据捕获，文件传输和日志记录：

在记录日志文件时你可以有很多选项，你可以将发射记录到日志，可以将接收记录到日志，也可以将两者都记录到日志文件。你也可以对已存在的日志文件选择追加和重写记录方式。使用 `autobin' 模式，你可以收发二进制文件。

远程 Shell 操作：

你可以为远程用户提供一个 SHELL,这样的话远程用户就可以运行你的计算机上的程序，你也可以运行一个程序并把它的输入、输出定位到一个通道上，这样远程用户也可以使用它。

将设备重定位成一个通道：

TNT 允许你将输入/输出从通道重定位到一个设备，比如说调制解调器。

变音：

如果需要的话 TNT 可以提供变音功能。

远程模式：

如果被允许的话，远程用户可以自己发出执行命令。

套接模式：

你可以配置 tnt 让它打开一个套接，这样的话，用户可以通过套接字的形式来操作你的机器，就象他真正坐在你的机器前一样。这种做法在下面的情况下很有用：连接你的收发信机的 Linux 机器在以太网的一端，在以太网另一端的终端就可以通过这种做法使用你的收发信机。

新的版本将支持 Linux 的 AX.25 内核代码。

DPBOX 是一个全功能的 BBS 软件，具有如下特点：

  • 支持 F6FBB, W0RLI, TheBox 支持的协议。

  • 使用 BBS 数据压缩以节省硬盘空间，上传和下传的时间。
  • "只抓取即时信息" 节省 BBS 频率，缓和网络通信量。
  • PACSAT 类型的布告板广播发射机和接收机。
  • 最大可同时支持 200 个用户。

当前开发状态

Alpha 版, 工作正常。

1997年4月1日

如果需要 AX.25 连接的话，网络内核必须支持 AX.25 协议。

因为 talk 协议并不是标准协议，所以不能实际上说明什么。但来自 NOS 的 ttylink 协议看起来象个标准协议，至少对于无线电爱好者来说，它能通过提供键盘到键盘的 TCP/IP 连接。本程序满足了 TTY 终端到 Linux 机器的连接。

从何处可取得本软件。

你可以从下面的站点取得 ttylinkd 的最新版本： sunsite.unc.edu 或 ftp.ucsd.edu。

在 Linux 的 AX.25 工具包里有一个稍微改进的版本。

使用许可/版权

GNU 公共许可，版本：2.0
