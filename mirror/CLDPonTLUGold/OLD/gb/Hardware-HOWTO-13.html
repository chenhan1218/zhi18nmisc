13. 硬碟机

所有的硬碟机应该都是被支援的，如果控制卡有被支援的话。

(引述自 SCSI HOWTO) 所有直接存取的 SCSI 磁碟机，不管磁区块的大小是 256, 512 或 1024 bytes 都可以使用。其他的磁区块大小则不能使用。 (注:通常这个问题可以使用MODE SELECT 的SCSI 命令以改变磁区块和/或磁区大小的方式来修正它)

大容量 IDE (EIDE) 磁碟机在新的 kernel 下工作的好好的。受限於 PC BIOS 的设计，开机磁区必须放在 前面的 1024 cylinders 磁柱□。

有些 Conner CFP1060S 磁碟机在使用 Linux 及 ext2fs 时会有问题。 在 e2fsck 时会发生 inode 错误的症状外，它还会搞坏档案系统。 Conner 已放出了一个轫体更新来修正这个问题，请打 1-800-4CONNER (US) 或 +44-1294-315333 (Europe)和 Conner 连络。 你打去後应该已有一个好用的微码版本了吧 (找到磁碟标□ 9WA1.6x)。

有些 Micropolis 磁碟机在和 Adaptec 及 BusLogic 介面卡使用时会有问题，如果你遇到问题时请和磁碟机制造商连络 以取得轫体升级。

13.1 不被支援的

下面提到的硬碟机并不被 Linux所支援。你可以找到臭虫报告。

  • NEC D3817, D3827, D3847
    "这些磁碟机有点非-SCSI-2 相关的，在 Mode Sense Page 3□有提到。在 Mode Sense Page 3 所有的 NEC D38x7 磁碟机会回报它们的磁区容量为零。NEC 磁碟机是我们第一次遇到磁区容量回报为零 的磁碟机品牌。不幸的，在 Mode Sense Page 3 的那个栏位是无法更改的，因此没有办法来升级 D38x7 磁碟机的轫体以便修正这个问题。"
