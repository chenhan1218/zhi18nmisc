1. 简介

什麽是 Linux?

Linux 是一个 POSIX 作业系统的独立实作，有著 SYSV 及 BSD 扩充，是从无到有整个写出来的(这表示它看起来及运作上就像 Unix)． 它里面没有专利独占的程式码．Linux 在 GNU Public License 下可自由地散布．

它可在 IBM PC 相容机器，有著 ISA 或 EISA(包含了 local bus 的变种 VLB 及 PCI)及 386 或更高级的微处理器上运作． 除此之外，它也能在具有 MMU 的某些 Amiga 及 Atari 电脑上执行． 这是说具有外部 MMU 的 68020, 68030, 68040, 或 68060． Digital Alpha 的支援现已经很稳定．Red Hat 及 Craftworks 有 Alpha 版的 Linux 套件． Sparc 的支援现也很稳定．Red Hat 有 Sparc 版的 Linux 套件． PowerPC 的支援现在发展多平台版，包括了 Nubus 及 PCI Macintosh, Motorola Powerstack, IBM 830 和 850, 以及其它平台． ARM, StrongARM 及 MIPS 的支援现已完成许多阶段，但仍不实用(hold your breath)．请见 comp.os.linux.announce．

这些平台支援的更多技术资讯请参见 Linux INFO-SHEET，确实的硬体需求参见 Hardware Compatibility HOWTO．

Linux 的核心是由 Linus Torvalds < Linus.Torvalds@.Helsinki.FI> 以及其它志愿者所撰写． 大部份在 Linux 上执行的程式是一般 Unix 的免费软体，许多是由 GNU 计画来的．

The Linux INFO-SHEET

上面有 Linux 更多的技术资讯．包括不同平台上的资讯指引，特徵列表，如何取得 Linux，及更多的资讯．

The Linux HOWTOs

有点像 FAQ，然而不是回答常见的问题，而是解释如何做一般的工作，像是整理一份 Linux 的发行版本，建立 Linux 的列印服务，建立基本的 UUCP □送等等． 所有 HOWTOs 的最新版本可见 http://sunsite.unc.edu/LDP/HOWTO/HOWTO-INDEX.html 或 ftp://sunsite.unc.edu/pub/Linux/docs/HOWTO/． 其它具有即时拷贝的 HOWTOs 文件站台有 ftp.cc.gatech.edu 及 tsx-11.mit.edu．

除此之外，还有一些短的，任意格式的文件叫 "mini-HOWTOs"． 这些文件涵盖了较特殊的主题，像是 BogoMIPS 或 Color-ls． 它们可从 ftp://sunsite.unc.edu/pub/Linux/docs/HOWTO/mini/ 及 http://sunsite.unc.edu/LDP/HOWTO/HOWTO-INDEX.html 取得．

Linux 新闻群组

有许多给 Linux 的专门新闻群组． 如果你使用 Linux，经常注意 comp.os.linux.announce 是个好主意． comp.os.linux.announce 现由 Lars Wirzenius 所管理． 要在此讨论区上陈述意见，写信到 linux-announce@news.ornl.gov． 你也可以将 comp.os.linux.announce 的问题直接向 Lars Wirzenius 陈述，地址是 wirzeniu@kruuna.helsinki.fi．

comp.os.linux.announce 是一个受管制的新闻群组，用来发布关於 Linux 的消息(新程式，错误修正等等)．

comp.os.linux.answers 是一个受管制的新闻群组，用来发表 Linux FAQ, HOWTO 文件及其它的文件用的．

comp.os.linux.setup 是一个未管制的新闻群组，用来讨论建立 Linux 系统所遭遇的问题．

comp.os.linux.admin 是一个未管制的新闻群组，用来讨论 Linux 系统的管理．

comp.os.linux.development.system 是一个未管制的新闻群组，专门用来讨论 Linux 的核心发展． 唯一可以在那里讨论的应用程式发展问题是与核心密切相关的． 所有其它的发展问题多半是一般的 Unix 发展问题而应被放到 comp.unix group，除非它们是与 Linux 有关的应用程式问题，这种情形的话问题应放到 comp.os.linux.development.apps．

comp.os.linux.development.apps 是一个未管制的新闻群组，专门用来讨论 Linux 相关的应用程式发展． 它不是用来讨论如何取得 Linux 的应用程式的，也不是用来给那些想看看 Linux 应用程式的人的讨论场所．

comp.os.linux.hardware 是给 Linux 相关的硬体问题用的．

comp.os.linux.networking 是给 Linux 相关的网路发展及建立的问题．

comp.os.linux.x 是给与 Linux 有关的 X Windows 的问题．

comp.os.linux.misc 是一个未管制的新闻群组，用来给不属於任何其它方面的 Linux 讨论．

一般来说，不要在 Linux 新闻群组上重复发表(crosspost)文章． 唯一适合重复发表的是偶而发布在一未管制的群组及 comp.os.linux.announce 的文章． 将 comp.os.linux 分割为许多群组的主要原因是为了降低每一群组的文章流量． 不遵守这项规则的话将会遭致他人毫不留情的抨击…

其它的新闻群组

不要假设你所有的问题都适合於 Linux 新闻群组只因为你在跑 Linux． 你的问题其实是关於任何 Unix 或类 Unix 的 shell 程式写作吗? 那麽试著到 comp.unix.shell 上去问． 它是关於 GNU Emacs 的吗? 那麽试著到 gnu.emacs.help 去问． 还有，如果你不知还有那些其它的群组可问，只是认为应该是这里了，用有礼的态度试问是否还有更适合你问题的讨论群组． 至少这些群组 comp.unix.{questions,shell,programming,bsd,admin}，以及 comp.windows.x.i386unix 应该是对 Linux 使用者有帮助的．
