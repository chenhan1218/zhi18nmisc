3. 文件相关问题

3.1 何处可取得 CLDP 的文件?

答: CLDP 目前以两种方式提供文件。一种是放在 WWW 伺服器上，可以浏览器直接在线上阅读的 HTML 格式，另一种是以压缩格式存放在 FTP 站台，可下载後解开阅读。 其中包括纯文字，HTML, SGML 等格式，皆放在对应的子目录下。所有的文件都同时提供 BIG5 及 GB 编码两种格式。 所有 WWW 及 FTP 皆有多个 映射站台。请选择离您最近的站台使用以获得较佳的网路传输效率。

3.2 如何一次取得所有 CLDP 的文件?

答: CLDP 所有的 FTP 映射站台都含有 tarball 的压缩档，里面便包含了所有的中译文件。 除此之外，各种文件格式都提供了 rpm 包装。如果您的系统是 rpm-based, 直接抓这些 rpm 包装的档案来装会较方便。 其中 HTML 的包装，安装完之後可以直接用 

lynx http://localhost/CLDP/
来观看(如果您也安装了 HTTP 伺服器的话)。

事实上，某些发行套件(distribution)已经包含 CLDP 文件在内了。 据我所知，这包括了 Red Hat, TurboLinux, 以及 Debian 等。 所以很有可能您根本不必另外装，这些文件已经在您的机器里面了。

如果您知道还有那些发行套件收录了 CLDP 文件，请告诉我。

3.3 CLDP 新的网页采用 JavaScript 是否对 lynx 不友善?

答: 否! CLDP 的新网页设计之初已将 lynx 之类的文字模式浏览器考虑在内。 因此像 lynx 之类的浏览器皆可正常的欣赏，没有任何障碍。

JavaScript 提供了 CLDP 网页较佳的导览功能。您可随时选择 关闭 或 打开 此功能。

3.4 我想提供站台映射(mirror) CLDP 的文件，该怎麽做?

答: 如前一问题所述，CLDP 的文件分为线上及离线阅读两种形式，也就是 WWW 及 FTP。 因此您先要弄清楚您想映射的是那一种，或是两者皆要。要映射全部内容约需 80MB 的硬碟空间。 除此之外，一旦您的映射站台对外开收，可能涌入大量人潮。因此您的站台必须有足够的对外频宽，并且机器须足够的稳定与强大。 这些是您在决定映射 CLDP 文件前所需考虑清楚的事项。 还有，您必须会使用映射的程式，如 mirror, wget 等。请不要写信问我如何使用这些程式。

如果您确定要做为映射站台，请来写信告诉我您想映射的文件形式，以及关於您站台的一些基本资讯，包括地理位置，机器资料，以及网路频宽等。 请提供足够的资讯，以便我决定是否能成为一可靠的映射站台。

如果您要映射 WWW 站台，您必须在伺服器上做下列这定(以 apache 为例):

  1. 在 mime.types 中加上这行: 
    application/x-javascript        js
  2. 在 srm.conf 加入这几行，将结尾为 .shtml 的档案设定为 Server Site Included (SSI): 
    # To use server-parsed HTML files
AddType text/html .shtml
AddHandler server-parsed .shtml
  3. 将 index.shtml 加到 srm.conf 的 DirectoryIndex 中。

至於映射 FTP 站台就比较简单。在下载完档案後开放权限即可。

一旦您的映射站台可正常运作并对外开放，请务必回信告诉我。我会在 CLDP 网页上为您加一连结，方便大家利用。还有，别忘了持续更新您的映射站台。至少一个礼拜更新一次。
