3. 对本文件的提案

我鼓励大家来维护一个 Linux 的套件或是邮购服务, 将它们的服务资讯提供到本文件中. 这是件容易及饶富趣味, 并且也是免费的广告. 这份文件它被贴在很多地方及归档起来(请看下一个章节就能了解)

3.1 提案内容的类型

我们对下列的提案内容是有兴趣的:

  • 可经由匿名的 FTP 站台, UUCP, 或邮购的方式取得的一个 Linux 完整套件. 我们的意思是经由任何的一组 `完整套件'软体就可以一从头就可用来建造成一个完整可用的 Linux 系统.
  • 仅能透过邮购方式购买到的各领域阶层产品及独自存在的软体套件. 假如您的软体套件是经由匿名的 FTP 站台取得时, 这表示是有找得到它的可能性. 如果只能经由邮购的方式取得的话, 那表示它包含商业性的一些东东, 诸如 Motif 和一些移植到 Linux 作业系统的商业版软体.
  • 任何可经由邮购方式购买得到的相关 Linux 特性 的商品, 像是 Linux 的特定书籍和文件, T-shirts 和各式各样的用具. 注意: 我们关心的是在後面这些, 它是有限制的-- 这是一份有关 Linux 发行套件的 HOWTO , 而不是一份讨人喜爱的 Linux 垃圾目录. 所以凡是任何您要销售的东东必须为 (a) 非常有用的, (b) 有创意的及 (c) 非常有趣的, 是这样的话才把它列到您的提案里.

假如为这些服务及阶层性产品提供的提案的数目很大的话, 我将为这些项目做一份分类式的 HOWTO(separate HOWTO).

3.2 如何提出

要提出一份讯息登录(entry)到这份 HOWTO 时, 请以下列的表示方式寄给我
esr@snark.thyrsus.com. 这个格式并不是一成不变的(machine-parsable); 在任何的讯息□围内您可作出任何您希望的长度, 但我希望每份登录都能保持在 50 行以下即可.

名称(Name):

套件或服务的名称

套件发行者(Distributor):

公司名称或连络人的姓名等等/谁在建造或维护这个套件或服务. 应该包括邮件地址, email , 电话等联络讯息, 假如可以的话也提供全球资讯网上的参考资讯.

供应者简介(Provider's Description):

您所提供这个套件或服务的叙述. 假如它是一个软体套件, 请包括一些相关讯息像是一个软体所应具备的版本号码, 安装的一般说明及设备需求等.

网路存取位址(Internet Access):

在网路上哪里可以取得您的套件及服务; 请提供代表性的全球资讯网或是 FTP 网站的网址.

订购方式(Ordering):

假如此套件及服务是适用的话, 那要如何订购呢? 这些应包括价钱, 运送资讯及付款的方法等等.

支援服务(Support):

有关支援服务的条件及契约等资讯.

杂记(Miscellaneous):

其它所有您发现有关的事项或讯息.

最新版本发表日期(Last Freeze Date):

当前版本的最後停留日期. 也请您预估更新的频率.

3.3 提案的指导原则

您的登录讯息请尽可能的简短. 假如您须要包含很多的资讯在里头的话, 请提出能在此套件上让大家藉由 FTP 或 mail 的方式取得更多资讯的说明办法; 这些登录讯息仅意谓著它如何在您的支援服务或套件中何处寻找资讯的 指示器 而已.

假如您要提供超过一个以上的套件或服务, 请为每一个使用 分类式的讯息登录(separate entries)方式.

为了使文件简洁扼要, 我如有发现任何不相干的资讯或是讯息过度地冗长, 我将会修改您的登录讯息. 否则内容就保持一样不加以变动.

当您制作一份发行套件指引(Distribution-HOWTO)的提案时, 您无条件的同意许可我使用这份登录讯息到其它的东东上去, 像是引用 LDP 的书籍和其它的线上文件. 举例来说, 如从 Distribution-HOWTO 上拮取的资讯所编成出版的书籍. 假如您不希望我除了这份 Distribution-HOWTO 之外把您的登录讯息包含到别的东东里面时, 请您告知我.
