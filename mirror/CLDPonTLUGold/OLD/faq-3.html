3. 文件相關問題

3.1 何處可取得 CLDP 的文件?

答: CLDP 目前以兩種方式提供文件。一種是放在 WWW 伺服器上，可以瀏覽器直接在線上閱讀的 HTML 格式，另一種是以壓縮格式存放在 FTP 站台，可下載後解開閱讀。 其中包括純文字，HTML, SGML 等格式，皆放在對應的子目錄下。所有的文件都同時提供 BIG5 及 GB 編碼兩種格式。 所有 WWW 及 FTP 皆有多個 映射站台。請選擇離您最近的站台使用以獲得較佳的網路傳輸效率。

3.2 如何一次取得所有 CLDP 的文件?

答: CLDP 所有的 FTP 映射站台都含有 tarball 的壓縮檔，裡面便包含了所有的中譯文件。 除此之外，各種文件格式都提供了 rpm 包裝。如果您的系統是 rpm-based, 直接抓這些 rpm 包裝的檔案來裝會較方便。 其中 HTML 的包裝，安裝完之後可以直接用 

lynx http://localhost/CLDP/
來觀看(如果您也安裝了 HTTP 伺服器的話)。

事實上，某些發行套件(distribution)已經包含 CLDP 文件在內了。 據我所知，這包括了 Red Hat, TurboLinux, 以及 Debian 等。 所以很有可能您根本不必另外裝，這些文件已經在您的機器裡面了。

如果您知道還有那些發行套件收錄了 CLDP 文件，請告訴我。

3.3 CLDP 新的網頁採用 JavaScript 是否對 lynx 不友善?

答: 否! CLDP 的新網頁設計之初已將 lynx 之類的文字模式瀏覽器考慮在內。 因此像 lynx 之類的瀏覽器皆可正常的欣賞，沒有任何障礙。

JavaScript 提供了 CLDP 網頁較佳的導覽功能。您可隨時選擇 關閉 或 打開 此功能。

3.4 我想提供站台映射(mirror) CLDP 的文件，該怎麼做?

答: 如前一問題所述，CLDP 的文件分為線上及離線閱讀兩種形式，也就是 WWW 及 FTP。 因此您先要弄清楚您想映射的是那一種，或是兩者皆要。要映射全部內容約需 80MB 的硬碟空間。 除此之外，一旦您的映射站台對外開收，可能湧入大量人潮。因此您的站台必須有足夠的對外頻寬，並且機器須足夠的穩定與強大。 這些是您在決定映射 CLDP 文件前所需考慮清楚的事項。 還有，您必須會使用映射的程式，如 mirror, wget 等。請不要寫信問我如何使用這些程式。

如果您確定要做為映射站台，請來寫信告訴我您想映射的文件形式，以及關於您站台的一些基本資訊，包括地理位置，機器資料，以及網路頻寬等。 請提供足夠的資訊，以便我決定是否能成為一可靠的映射站台。

如果您要映射 WWW 站台，您必須在伺服器上做下列這定(以 apache 為例):

  1. 在 mime.types 中加上這行: 
    application/x-javascript        js
  2. 在 srm.conf 加入這幾行，將結尾為 .shtml 的檔案設定為 Server Site Included (SSI): 
    # To use server-parsed HTML files
AddType text/html .shtml
AddHandler server-parsed .shtml
  3. 將 index.shtml 加到 srm.conf 的 DirectoryIndex 中。

至於映射 FTP 站台就比較簡單。在下載完檔案後開放權限即可。

一旦您的映射站台可正常運作並對外開放，請務必回信告訴我。我會在 CLDP 網頁上為您加一連結，方便大家利用。還有，別忘了持續更新您的映射站台。至少一個禮拜更新一次。
