如何翻譯 Linux HOWTO 或 mini HOWTO

  1. 首先，當然您要會讀寫中文，也要看得懂點英文啦! 不過也別緊張! 您不需要托福考六百分以上才能翻譯 Linux HOWTO。事實上大部份的英文都算簡單的。 因為，許多 HOWTO 作者的母語也不是英文呀!

  2. 閱讀一下 CLDP 常見問題與解答，很多您想知道的事情都已經寫在裡面了。

  3. 再來，選一篇您看起來有興趣，或有些研究的 HOWTO 或 mini HOWTO 文章為目標。 至於到底有那些 HOWTO 或 mini HOWTO 文章，請看 HOWTO INDEX。

  4. 確定您選擇的目標還沒有被人翻譯過。請參考翻譯中文件列表以及未翻譯文件列表。 您最好先與我聯絡一下以確定您要翻譯的文件還未被人"認領"。

  5. 所有的 HOWTO 文件版權都是屬於原作者的。為了禮貌，最好先取得原作者的翻譯同意權。這樣將來文件有更新時或許也會通知您。 我想絕大部份的作者應該都會欣然同意您翻譯他的文章。

  6. 如果您想翻譯的是 HOWTO 文件，必須寫成 SGML 文件，如此才能轉換為各種輸出格式。 所以您必須先了解什麼是 SGML。請先參考 SGMLtools User's Guide。 您不真正需要了解到底什麼是 SGML (其實我也不是很了解…:p)，但您應該要能看得懂 SGML 文件中的 SGML 標籤，如此在翻譯時才能分辨何者是文章內容，何者是 SGML 標籤。 至於 mini HOWTO 則有純文字以及 SGML 兩種格式。如果是用 SGML 的，當然建議您也使用 SGML。下面所說的，都是假設您要翻的是 SGML 文件。

  7. 取回您想翻譯文件的 SGML 原始文件。最簡單的辦法就是寫信給我，我會寄給您。

  8. 如果您要自行取得也可以。但是請務必要取得最新版本的文件，大部份人都不會想看已經過時的文章。 您可在下列地點取得:

    • Metalab 原始站
    • 國內映射站: 中山 FTP | 中正 FTP

  9. 如果您不確定取回的文件是否為最新版，可與 HOWTO INDEX 上的更新日期比較。 某些文件還會註明最新版本的發表站台，您也可從該處取回。

  10. 再來就是用力的翻譯啦! 不過還是有幾點請您注意:

    • 請儘量保持原始 SGML 文件的格式，包括章節，順序，甚至一些特殊標籤(如 <tt>, <em>)，URL 等等。 這點其實很容易辦到。您只要將原始 SGML 文件中的英文翻成中文，其它的 SGML 標籤都別去動它就好了。
      例如原文為:
      For dialin and dialout use, you <EM/have/ to set up your modem a certain way (again, using <TT/AT/ commands on your modem):
      翻譯為:
      要用來接受撥接以及撥號出去的話，那麼你<EM/必須/確實地設定你的數據機(再一次對你的數據機使用 <TT/AT/ 指令群):

    • 您不需將翻譯的 SGML 文件排版整齊，因為這沒有意義。將 SGML 轉為其它格式時會自動為您排版。 不過仍有一點要小心，就是最好不要在中文句子中間換行，否則轉為 HTML 格式時它會在中間加上空格。 例如若您在 SGML 中寫: 

      這是一個很
長的句子。

      轉成 HTML 後可能會變成

      這是一個很 長的句子。

      這樣變得不好看。因此換行的地方最好在句末。

    • 在中英文夾雜的句子中，英文字與中文間前後最好加上空格比較好看。例如:

      Linux,FreeBSD與Solaris都是PC上的UNIX作業系統。

      看起來就沒有

      Linux, FreeBSD 與 Solaris 都是 PC 上的 UNIX 作業系統。

      這樣好看。當然，這純屬個人意見，您無須遵守。

    • 人名，地名，公司名等專有名詞以不翻譯為原則，直接以原文寫出較好。

    • 其它的相關問題也請見 CLDP 常見問題解答 的說明。

  11. 翻譯完成後請先測試您的 SGML 原始文件文法是否正確，可否順利編譯。您可安裝 SGMLtools 中文套件 zh-sgmltools. 請參考 CLDP 常見問題解答 的說明。

  12. 測試無誤後請將 SGML 文件寄給我。我會將它轉換為各種格式並放在適當位置。 當然，不會忘了在翻譯列表上加上您的大名。

  13. 之後，每當您翻譯的文件有了新版本，我會主動將更新版本寄給您。然而，請保留您翻譯過的英文 SGML 原始文件。 當您取得更新版的文件時，只要用 diff 指令就可很快找出更新的部份。
    diff -uN Foo-HOWTO.sgml.old Foo-HOWTO.sgml
    當您更新文件後記得再將它寄給我。

  14. 再來呢…? 再來就沒啦! 簡單吧? ^_^
