13. 硬碟機

所有的硬碟機應該都是被支援的，如果控制卡有被支援的話。

(引述自 SCSI HOWTO) 所有直接存取的 SCSI 磁碟機，不管磁區塊的大小是 256, 512 或 1024 bytes 都可以使用。其他的磁區塊大小則不能使用。 (註:通常這個問題可以使用MODE SELECT 的SCSI 命令以改變磁區塊和/或磁區大小的方式來修正它)

大容量 IDE (EIDE) 磁碟機在新的 kernel 下工作的好好的。受限於 PC BIOS 的設計，開機磁區必須放在 前面的 1024 cylinders 磁柱堙C

有些 Conner CFP1060S 磁碟機在使用 Linux 及 ext2fs 時會有問題。 在 e2fsck 時會發生 inode 錯誤的症狀外，它還會搞壞檔案系統。 Conner 已放出了一個軔體更新來修正這個問題，請打 1-800-4CONNER (US) 或 +44-1294-315333 (Europe)和 Conner 連絡。 你打去後應該已有一個好用的微碼版本了吧 (找到磁碟標韱 9WA1.6x)。

有些 Micropolis 磁碟機在和 Adaptec 及 BusLogic 介面卡使用時會有問題，如果你遇到問題時請和磁碟機製造商連絡 以取得軔體升級。

13.1 不被支援的

下面提到的硬碟機並不被 Linux所支援。你可以找到臭虫報告。

  • NEC D3817, D3827, D3847
    "這些磁碟機有點非-SCSI-2 相關的，在 Mode Sense Page 3埵陷ㄗ魽C在 Mode Sense Page 3 所有的 NEC D38x7 磁碟機會回報它們的磁區容量為零。NEC 磁碟機是我們第一次遇到磁區容量回報為零 的磁碟機品牌。不幸的，在 Mode Sense Page 3 的那個欄位是無法更改的，因此沒有辦法來升級 D38x7 磁碟機的軔體以便修正這個問題。"
