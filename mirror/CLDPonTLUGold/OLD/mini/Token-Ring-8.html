8. 套件特有安裝

8.1 Slackware 96

如果你的系統未執行過 "netconfig" ,請現在執行. 設定值如同系統是處於 Ethernet(乙太網路) 中.

編輯 /etc/rc.d/rc.inet1 指向 tr0 device(設備)而不是eth0 device(設備)

你應該會看到有一行像這樣 

/sbin/ifconfig eth0 ${IPADDR} broadcast ${BROADCAST} netmask ${NETMASK}
把這行改成為 
/sbin/ifconfig tr0 ${IPADDR} broadcast ${BROADCAST} netmask ${NETMASK}

8.2 RedHat 4.0+

在 RedHat Linux 中有一個很棒的 Xwindows 控制台可以設定 modules(模組) 和 devices(設備). 如果你在安裝 RedHat 時就已經裝上了 token ring 網路卡,那麼你應該已經有了 token ring 的模組支援了. 試試以下指令: 

modprobe ibmtr
看看你得到甚麼. 如果你的網路卡被支援,那麼很可能你已經能使用它了.

設定介面最好用的是 control-panel(控制台). 首先你會想用 kernel module control(模組核心控制) 並指示它載入 token ring 這個新的網路設備. 它應該會在開機時自動載入 ibmtr 這個驅動程式. 然後你會想跳過網路組態並增加新的設備,tr,並給它一個 ip address,等等.

你可以重新開機並在 RedHat Linux 4.0+ 上使用 Token Ring 網路卡了.

請留意關於 RedHat 的說明部份是靠腦中的印象寫的. 因為我有一段時間沒有修改過我的 RedHat 系統了,現在這是憑記憶寫的. 不論如何,它們可以達成你的目的.
