2. 認識 Linux 使用者協會

2.1 Linux 系統的特性

想要充分瞭解 Linux 使用者協會 (LUG) 在 Linux 運動當中扮演的角色， 就要先從認識 Linux 之不同於其他作業系統的特性開始。

從作業系統的角度來看，Linux 確實是一個功能強大、效率高超的軟體； 然而真正要瞭解 Linux 為何能夠眾不同，就 必須從軟體研發的觀念與模式這個角度來看。 Linux 是一個自由作業系統: 它受到 GNU 一般公共許可証 (GNU General Public License) 的保護，其程式原始碼將永遠公開。 來自世界各個角落的程式設計師在 Linux Torvalds 及其他主要研發者的領導下共同參與 Linux 的研發工作。 這一個全球性的運動背後並沒有一個官僚體系或是單一的指導組織在主導它的發展方向. 這樣的特殊背景單就技術層面上而言，對 Linux 作業系統本身的品質是一種保障； 但是在其他方面或許會造成人力資源無法充分發揮的結果， 例如在推廣、使用者培訓等方面都無法達到良好的效果。

2.2 Linux 系統為何與眾不同?

Linux 系統獨特的鬆散研發結構不太可能改變；但這也是一件好事。 Linux 之所以有良好的品質正因為研發者可以自由發揮， 自由的程式設計師自然是愉快的設計師, 也正是有效率的設計師。

另一方面，對於一位非程式設計師的一般使用者而言，這種狀況卻會帶來一些困擾。 有誰可以教導 Linux 系統的使用？ 如果有問題應該要向誰求助？ 如何知道 Linux 最適用的場合？

各個區域性的 LUG 往往是解決上述問題的資訊來源， 這也就是為什麼 LUG 在這個全球性的 Linux 運動當中扮演舉足輕重的地位。 在許多方面， LUG 都扮演了類似一般大型跨國企業的地區性分公司的角色。

Linux 與眾不同之處正在於它沒有一個中央集權式的指導組織來分配企業資源、提供使用者培訓及技術支援，

但這也免除了一個中央控制組織所帶來的沉重負擔。 這些工作有許多其他不同的管道來達成，例如網際網路、技術顧問、加值零售商 (VAR)、技術服務公司、大專院校等等。 然而在全球各地越來越多的地方，區域性的 LUG 已逐漸也開始提供這些服務。

2.3 Linux 使用者協會為何與眾不同?

電腦使用者協會並不是一個新鮮的玩意兒 (至少在美國行之已久)。 不但如此，它在個人電腦的歷史當中還扮演了相當重要的角色。 事實上，個人電腦的風行有一大半是由於電子組件、火腿族、及其他各種使用者協會的需求所致； 當然也因為市場上對於低價電腦的需求進而產生了展銷會、跳蚤市場等現象。 當然諸如 IBM 此類大公司最後終究體認到個人電腦業有利可圖； 但是使 PC 市場活絡的原動力還是來自使用者的需求、由使用者所發起、為解決使用者的問題而產生的。 在美國，多數使用者協會每下愈況， 知名的 波士頓電腦協會面臨的財政危機曾廣受媒體的注意； 而在美國整體而言，大多數大型的個人電腦使用者協會都面臨了真正會員人數減少的窘境。 美國的使用者協會在全盛時期將他們的資源集中在出版刊物、維護共享軟體及程式庫磁片、舉行會議、 活動、乃至經營 BBS 站等方面。 然而隨著網際網路的普及, 過去由這些傳統的使用者協會提供多項的服務卻逐漸轉移到 CompuServe、AOL 和網路上。

另一方面 Linux 的興起卻正好與網際網路的普及同步發展。 隨著網路深入每一個角落，Linux 也因而受惠； 因為網際網路為 Linux 運動帶來了越來越多的使用者、研發者與經銷商。

網路的普及對傳統的個人電腦使用者協會極為不利； 但對 Linux 而言卻正是成長茁壯的搖籃， 因而造成了對 Linux 使用者協會的強烈需求。 為了要說明 Linux 使用者協會與傳統的使用者協會究竟有什麼本質上的不同， 請讀者諸君看一個有趣的現象就可以知道了： 傳統的使用者協會必須嚴格監控它的成員在會議、活動當中是否有非法拷貝的行為。 使用者協會不會允許這樣的行為發生，而且這麼作也有他的道理； 然而非法拷貝的行為總是會發生。

反觀在 LUG 的會議或活動當中, 整個 Linux 社會對資訊分享的觀念完全不同； LUG 不但不阻止拷貝行為，反而還大力鼓吹。 事實上，有不少例子在在說明許多傳統的使用者協會很難接受 Linux 可以自由拷貝這樣的事實. (譯註: 這其實要歸功於 自由軟體基金會所推廣的 GPL, 因為 Linux 本身及其上的許多應用軟體都受到這個版權聲明的保護， 人人都可以自由拷貝，但是沒有人可以剝奪下游使用者的權力。)

2.4 結論

區域性的 LUG 對於 Linux 運動的發展有相當大的影響。 基於 Linux 獨特的性質， LUG 應該要扮演類似 IBM、微軟、昇陽等等大公司他們的區域性分公司所扮演的角色。 LUG 應該要提供 Linux 用戶教育訓練、技術支援、協調顧問群、推廣 Linux 的使用等等， 乃至與新聞媒體建立良好的關係。
