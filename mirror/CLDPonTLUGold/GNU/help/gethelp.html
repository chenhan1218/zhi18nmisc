如何得到 GNU 軟體幫助

自由軟體基金會不提供技術支援，我們的任務是開發，維護和保護 自由軟體。 我們讓其他人通過提供支援來謀生。我們發現現在程式設計師和醫生律師一樣提供服務，醫學和法律知識是自由傳播的，但是實踐者為服務收費。 所以請不要電郵或致電基金會辦公室來尋求技術支援。

您可以通過許多途徑獲得 GNU 計劃文件，這些文件能回答許多您的問題。

我們開通了許多 GNU 軟體郵件列表。這些以「help-」開頭的郵件列表是向社群求助的列表。 我們也提供一個人工維護的 GNU 郵件列表表。如果您不能找到某一實際 GNU 程式的郵件列表，請使用 mailto:help-gnu-utils@gnu.org郵件列表。

GNU 服務目錄是一個提供支援和其他咨詢服務人們的列表。它也收藏在 GNU Emacs發布的「ect/SERVICE」 文件中和 GNU FTP主機 的「/pub/gnu/GNUinfo/SERVICE」中。您可以聯系我們得到一份拷貝或者列於上面。

如果您在GNU軟體中找到一個缺陷，我們希望知道。因此，我們也為 GNU 軟體的 BUGS 維護 郵件列表。即使有郵件列表未列出，典型地 <bug-PROGRAM@gnu.org> 地址將到達BUG報告的正確地方（通過一個別名）。 所以如果您在 GNU 程式的找到一個 BUG，請試著按這個方法報告。

當我們收到一個 BUG 報告，我們通常試圖去修正該問題。盡管我們的 BUG 修正工作看起來象是個人協助， 但是實際上不是。他們是准備一個新的改進版本的一部分。我們可能會給您一個 BUG 的補丁以便您能幫助我們測試該修正和驗證其質量。如果您的 BUG 報告沒有引發來自我們的解決，您仍然可以從閱讀 我們 BUG 報告郵件列表的其他用戶得到一個。否則，使用 GNU 服務目錄。

請不要為了安裝軟體或學習使用它而向我們求助--但是務必告訴我們安裝腳本如何失敗或文件哪裡不清楚。

最後，請注意現在許多公司都再發布 GNU 軟體，通常是作為 GNU/Linux 發布的一部分。當您發現您安裝的 一個 GNU/Linux 發布的 GNU 程式的 BUGS，首先直接向發布者報告這個 BUG 會有用，而不是報告給我們。有時，發布者修改了 GNU 軟體（因為他們可以自由地這樣做！）或者他們運行較老的版本。因此，他們可能是找到在某一特定發布中存在 BUG 的最佳人選。
